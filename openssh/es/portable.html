OpenSSH portable

El equipo de desarrollo de OpenSSH produce una versión para el proyecto OpenBSD muy pequeña, segura, y fácil de mantener. El Equipo de Portabilidad de OpenSSH coge esa versión pura y le añade código de portabilidad, para que OpenSSH pueda funcionar en otros sistemas operativos (desafortunadamente, y debido a que OpenSSH usa autenticación, tiene muchas diferencias en los distintos sistemas operativos Unix).

OpenSSH está diseñado para funcionar en los siguientes sistemas operativos Unix:

  • AIX
  • HP/UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • MacOS X
  • ... y más que van siendo añadidos continuamente.

La versión portable de OpenSSH sigue el desarrollo de la versión oficial, pero las versiones no están sincronizadas. Las versiones portables están marcadas con una 'p' (v.g. 2.3.0p1). El código fuente oficial para OpenBSD nunca se usará el sufijo 'p', sino que se incrementará el número de la versión cuando llegue a «puntos estables» en el desarrollo.

Los siguientes ficheros describen los esfuerzos llevados a cabo para su desarrollo por parte del equipo de portabilidad de OpenSSH:

  • Fichero de cambios en portabilidad ("ChangeLog")
  • Instrucciones de instalación
  • Preguntas frecuentes
  • Versiones de prueba diarias

Desarrolladores

Las versiones extremas de prueba están disponibles desde aquí. Las versiones portables de OpenSSH también están disponibles mediante CVS anónimo. Para acceder a éstas debe usar las siguientes órdenes: 

cvs -d :pserver:cvs@bass.directhit.com:/cvs login
cvs -d :pserver:cvs@bass.directhit.com:/cvs co openssh_cvs
(pulse intro cuado lo pida la contraseña). Pedimos a los desarrolladores que lean y sigan estas guías cuando entreguen código. Los parches se deben enviar en formato diff unificado.
