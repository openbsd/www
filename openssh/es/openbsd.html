OpenSSH para OpenBSD

Ya que el desarrollo de OpenSSH lo lleva a cabo el grupo de OpenBSD, OpenSSH va incluido en el sistema operativo básico desde la versión 2.6 de OpenBSD. Así pues, no hay nada que deba ser instalado o añadido a su sistema. Sin embargo, debido a la aparición de funcionalidades más recientes o de otros parches, es probable que desee actualizar la versión que se instaló con el sistema.

Puede obtener la última versión de OpenSSH desde las réplicas de ftp.

Las siguientes versiones de OpenSSH se encuentran disponibles para su instalación en máquinas con el sistema operativo OpenBSD:

  • OpenSSH 2.9.9: 25 de septiembre de 2.001

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-2.9.9.tgz

    OpenSSH 2.9.9 compilará perfectamente en un sistema OpenBSD 2.8 o 2.9 sin modificaciones: 

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.9.9.tgz
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Asegúrese de que exista la línea 'HostKey /etc/ssh_host_dsa_key' en su fichero de configuración sshd_config.

    Para extraerlo e instalarlo en un sistema OpenBSD 2.7:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.9.9.tgz
# cd ssh
commente 'CFLAGS+=-DHAVE_LOGIN_CAP' en sshd/Makefile, usando un carácter #.
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Nota: La compilación del código fuente anterior falla en versiones anteriores de OpenBSD debido a que el soporte para IPV6 en libc estaba incompleto.

  • OpenSSH 2.2.0: 31 de agosto de 2.000

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-2.2.0.tgz

    Para extraer el paquete e instalarlo en un sistema OpenBSD 2.7:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.2.0.tgz
# cd ssh
comente 'CFLAGS+=-DHAVE_LOGIN_CAP' en sshd/Makefile, usando un
caracter #
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Nota: Este código falla al compilar en versiones anteriores de OpenBSD, debido a que el soporte para IPV6 en libc estaba incompleto.
    Si va a instalarlo en OpenBSD 2.6, necesitará el siguiente parche:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd26_2.2.0.patch.

    Para extraerlo e instalarlo en un sistema OpenBSD 2.6:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.2.0.tgz
# patch -p0 < openbsd26_2.2.0.patch
# cd ssh
elimine el caracter de comentario # que precede a
'CFLAGS+=-DHAVE_LOGIN_CAP' en sshd/Makefile
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc

    Nótese que para usar OpenSSH 2.2.0 en OpenBSD 2.6 necesitará instalar los paquetes ssl27.tgz o sslUSA27.tgz de la biblioteca ssl en lugar de las versiones correspondientes para 2.6. De este modo se actualizarán sus bibliotecas ssl a una versión más nueva de OpenSSL que contiene nuevas llamadas requeridas por OpenSSH. Además, si también quiere usar SSH2 y aún no dispone de una clave DSA, tendrá que generar una mediante la orden 

    # /usr/bin/ssh-keygen -d -f /etc/ssh_host_dsa_key -N ''
