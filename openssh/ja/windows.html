Windows

以下の「フリー」 SSH が Windows マシンから OpenSSH と相互運用できるものとしておすすめです:

  • PuTTY は SSH1 と SSH2 プロトコルの実装です (日本語化パッチはこちら)。 PSCP という Windows 用の scp 風プログラムも使えます。

    PuTTY は (BSDライクな) MIT のライセンスで使用できます。

    「PuTTY は Win32 プラットフォーム用の Telnet および SSH の フリーな実装であり、基本的にイギリス在住の Simon Tatham によって 書かれ、メンテナンスされています」

  • TTSSH (SSH1) は SSH1 プロトコルのみの実装で、Robert O'Callahan の手によるものです。

    「TTSSH はフリーの Windows 用 SSH クライアントです。 これは Tera Term Pro の拡張 DLL として実装されています。 Tera Term Pro は非常によくできたフリーの Windows用 端末エミュレータ / telnet クライアントであり、そのソースも利用できます。TTSSH は Tera Term Pro の本来ある機能をまったく損うことなく、SSH の 機能を追加します。TTSSH もまたフリーでダウンロードでき、 そのソースもオープンソースライセンスで使えます。さらに TTSSH は 全部オーストラリアで開発されています (中略)。」

  • Cygwin (Windows 上で動く POSIX ソフトウエア)

    Cygwin を使えば Windows 上で OpenSSH (SSH1 および SSH2 プロトコル) が 走ります。移植版 OpenSSH を使っています。

  • MSSH

    Metropolitan State College of Denver で開発された MSSH は Windows 95 および Windows 98 上で SSH1 プロトコルをサポートしています。

  • Secure iXplorer GPL

    Secure iXplorer GPL は PuTTY の pscp.exe のグラフィカル フロントエンドです。

  • WinSCP

    WinSCP は Windows 上で動く sftp(1) および scp(1) プログラムです。 ここで使っている SSH プロトコルのコアは PyTTY をベースにしています。

  • FileZilla

    FileZilla は Windows 9x, ME, NT4, 2000 および XP 上で動く高機能の FTP クライアントです。 これは簡単に使えることと、なるべく多くの機能をサポートすることを目指して 設計されており、それでいて高速かつ安定に動作します。これは SFTP をサポートしています。 (訳注: 春山さんによる ファイル名 EUC<->SJIS 変換パッチ があります)

  • SSHWindows

    • OpenSSH for Windows はフリーのパッケージで、 Cygwin に含まれている最小の OpenSSH サーバおよびクライアントユーティリティを Cygwin のフルインストレーションなしで使うことができます。 これは SSH/SCP/SFTP をサポートしており、SSH 端末はおなじみの Windows コマンドプロンプトを提供しながら、SCP および SFTP 用には Unix/Cygwin スタイルのパス名を保持します。
