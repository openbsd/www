Portable OpenSSH

Normal OpenSSH development produces a very small, secure, and easy to mantain version for the OpenBSD project. The OpenSSH Portability Team takes that pure version and adds portability code so that OpenSSH can run on many other operating systems (Unfortunately, in particular since OpenSSH does authentication, it runs into a *lot* of differences between Unix operating systems).

The OpenSSH provided here is designed to run on the following Unix operating systems:

  • AIX
  • HP/UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • MacOS X
  • ... and more are being added all the time.

The portable OpenSSH follows development of the official version, but releases are not synchronised. Portable releases are marked with a 'p' (e.g. 2.3.0p1). The official OpenBSD source will never use the 'p' suffix, but will instead increment the version number when they hit 'stable spots' in their development.

The following files describe the development efforts of the OpenSSH portability development team:

  • Portability ChangeLog
  • Installation instructions
  • FAQ
  • Daily snapshots

Developers

Bleeding-edge nightly snapshots are available here. The portable releases of OpenSSH are also available by anonymous CVS. To access these you will need to issue the following commands: 

cvs -d :pserver:cvs@bass.directhit.com:/cvs login
cvs -d :pserver:cvs@bass.directhit.com:/cvs co openssh_cvs
(just hit enter when asked for a password). Developers are asked to read and follow these guidelines when submitting code. Patches should be send in unified diff format.
