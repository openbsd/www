Wie man die Dateien dieser Version bekommt:
  • Bestelle einfach eine CDROM über unser Bestellsystem.
  • Informationen über mirror Maschinen gibt es auf unsere FTP Seite.
  • Gehe in das pub/OpenBSD/3.1/ Verzeichnis auf einer der Mirror-Sites.
  • Lies den Rest dieses Dokumentes sorgfältig.
  • Wirf einen Blick auf die 3.1 Errata Seite auf der sich gegebenfalls eine Liste von Bugs findet, und wie man sie umgeht oder loswird.
  • Wirf einen Blick auf das detaillierte Logfile der Änderungen zwischen den Versionen 3.0 und 3.1.



Hinweis: Alle anwendbaren Copyrights und Danksagungen können in den passenden Quellcodedateien in src.tar.gz, srcsys.tar.gz, XF4.tar.gz, oder den Dateien gefunden werden, die via ports.tar.gz geholt wurden. Die Distributionsdateien die man braucht um mit Hilfe der ports.tar.gz Datei packages zu bauen sind aus Platzgründen nicht mit auf den CDROMs.

Was gibt es Neues

Dies ist eine unvollständige Liste von neuen Features und Teilsystemen, die in OpenBSD 3.1 integriert sind. Eine vollständigere Liste findet sich im changelog zu 3.1.

  • OpenSSH (Sowohl das SSH1- als auch das SSH2-Protokoll werden unterstützt) ist jetzt bei Version 3.0. Sichere Dateitransfers sollten jetzt mit dem stark verbesserten SFTP Subsystem gemacht werden, das sowohl einen SFTP Server als auch einen Client enthält.

  • Wie immer wurde die Dokumentation verbessert, und zwar besonders die man pages und die Web FAQ. Ein grösserer Teil der Webseiten ist jetzt in verschiedenen Sprache vorhanden.

  • Eine vollständigere und besser getestete Sammlung von "ports".

  • Über 1000 vorkompilierte und getestete packages.

  • Stark verbesserte Hardwareunterstützung im OpenBSD/sparc64 port, und zusätzlich jetzt auch Unterstützung für X11.

  • Zahlreiche Performance-Verbesserungen für den OpenBSD/macppc port und ausserdem beschleunigte X11 Server für einige Modelle.

  • Eine Menge Erweiterungen für unseren neuen Packet Filter, pf, inklusive Performance-Verbesserungen, oder der Möglichkeit auch andere Protokolle neben TCP, UDP oder ICMP, wie z.B. ESP zu filtern.

  • Ein Utility um pf-Regeländerungen abhängig vom User zu erreichen, typischerweise sinnvoll für Gateways, authpf.

  • Das im Kernel enthaltene Software-RAID (RAIDframe) entdeckt und konfiguriert jetzt automatisch die RAID-Geräte beim Booten. Dadurch kann das System seine root Partition direkt von einer automatisch erkannten RAID Partition mounten.

  • Wavelan bridging ist jetzt mit Prism-II basierten Karten möglich.

  • Das System beinhaltet die folgenden grossen Komponenten von Aussenstehenden:

    • XFree86 4.2.0 (und i386 enthält auch 3.3.X Server, damit Unterstützung für alle Chipsets gewährleistet ist)
    • gcc 2.95.3 (+ patches)
    • perl 5.6.1 (+ patches)
    • Apache 1.3.24, mod_ssl 2.8.8, OpenSSL 0.9.6b (+ patches), DSO Unterstützung
    • groff 1.15
    • sendmail 8.12.2
    • lynx 2.8.2rel.1 mit Unterstützung für HTTPS
    • sudo 1.6.5p2
    • ncurses 5.2
    • Latest KAME IPv6
    • KTH Kerberos 1.0.8
    • Heimdal 0.4e (+ patches)
    • OpenSSH 3.2

  • Viele Verbesserungen in Bezug auf Sicherheit und Zuverlässigkeit (siehe auch die roten Zeilen im kompletten changelog).

Wie man es installiert

Es folgen Instruktionen, die du in gedruckter Form haben würdest, wenn du anstatt einer alternativen Installationsmethode eine CDROM gekauft hättest. Die Anleitungen für FTP-Installationen sind recht ähnlich, die Anleitungen für die CDROM-Installationen wurden jedoch intakt gelassen, um zu zeigen, um wieviel einfacher alles mit einer CDROM gewesen wäre.

Bitte beachte auch die folgen Dateien auf den CDROMs, die noch weitere Details enthalten, um OpenBSD 3.1 auf deiner Maschine zu installieren:

  • CD1:3.1/i386/INSTALL.i386
  • CD1:3.1/alpha/INSTALL.alpha

  • CD2:3.1/macppc/INSTALL.macppc
  • CD2:3.1/vax/INSTALL.vax

  • CD3:3.1/sparc/INSTALL.sparc
  • CD3:3.1/sparc64/INSTALL.sparc64

    • Nur per FTP erhältich:

  • CD4:3.1/mvme68k/INSTALL.mvme68k
  • CD4:3.1/amiga/INSTALL.amiga
  • CD4:3.1/mac68k/INSTALL.mac68k
  • CD4:3.1/hp300/INSTALL.hp300

Hier gibt es kurze Installationsinformationen für Leute, die mit OpenBSD vertraut sind und den Befehl "disklabel -E" kennen. Wenn du irgendwie unsicher in Bezug auf die Installation bist, lies die entsprechenden INSTALL.* Dateien, die oben angegeben sind!

OpenBSD/i386:

    spiele mit den BIOS-Optionen herum und versuche, von CD zu booten. Die i386 Version von OpenBSD ist auf der CD1. Wenn dein BIOS kein Booten von CD unterstültzt, schreibe CD1:3.1/i386/floppy30.fs auf eine Floppy, und nimm diese als Boot-Medium.

    Benutze CD1:3.1/i386/floppyB31.fs für eine bessere Unterstützung von SCSI-Controllern oder CD1:3.1/i386/floppyC31.fs für bessere Laptop-Unterstützung.

    Wenn du OpenBSD neben anderen Betriebssystemen betreiben willst, solltest du unbedingt das Dokument INSTALL.i386 lesen.

    Benutze das "rawrite" Werkzeug, von CD:/3.1/tools/rawrite.exe um eine Floppy unter MS-DOS zu erstellen. Unter Unix einfach das dd(1) Utility benutzen. Das folgende Beispiel für dd(1) könnte "floppy", "rfd0c", oder "rfd0a" als Device benutzen. 

      # dd if=<file> of=/dev/<device> bs=32k

    Benutze ausschliesslich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen. Mehr Informationen über die Installation von OpenBSD/i386 gibts in der FAQ4.1.

OpenBSD/alpha:

    Am SRM prompt gibst du boot -fi 3.1/alpha/bsd.rd dka6 ein, wobei dka6 der Kurzname für das CDROM-Laufwerk ist (prüfe das mit show dev).

    Wenn du nicht von CDROM booten kannst, schreibe CD1:3.1/alpha/floppy31.fs oder CD1:3.1/alpha/floppyB31.fs (abhängig von deiner Maschine) auf eine Diskette und gib boot dva0 ein. In INSTALL.alpha gibt es noch weitere Details.

    Benutze ausschliesslich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen.

OpenBSD/macppc:

    Lege die CD2 in dein CDROM Laufwerk und drücke die C Taste während des Einschaltens, bis schliesslich die Anzeige OpenBSD/macppc boot zeigt.

    Alternativ kannst du am Open Firmware prompt boot cd:,ofwboot /3.1/macppc/bsd.rd eingeben.

OpenBSD/hp300:

    Lege die CD4 in dein CDROM Laufwerk und drücke die Leertaste während des poweron self-test. Wähle die boot Möglichkeit, die mit SYS_CDBOOT übereinstimmt.

    Alternativ kannst du über Netzwerk booten, indem du den Anweisungen in INSTALL.hp300 folgst.

OpenBSD/amiga:

    Erzeuge BSD Partitionen gemäß der Vorbereitungssektion in INSTALL.amiga. Mounte die CD4 unter AmigaOS als Device CD0: Danach führe folgendes CLI-Kommando aus: "CD0:3.1/amiga/utils/loadbsd CD0:3.1/amiga/bsd.rd".

OpenBSD/mac68k:

    Boote MacOS ganz normal und partitioniere deine Festplatte mit den passenden A/UX Konfigurationen. Dann packe die Macside Utilities aus dem Verzeichnis CD4:3.1/mac68k/utils aus und speichere sie auf deiner Festplatte. Benutze dann Mkfs, um die Dateisysteme auf den frisch eingerichteten A/UX Partitionen zu erzeugen. Den BSD/Mac68k Installer benutzt du danach, um alle Sets aus CD4:3.1/mac68k/ auf deine Partitionen zu kopieren. Und zum guten Schluss bist du nun soweit, um den BSD/Mac68k Booter mit dem Ort zu konfigurieren, an dem sich dein Kernel befindet und das System zu rebooten.

OpenBSD/mvme68k:

    Du kannst entweder ein bootbares Band erzeugen oder über Netzwerk booten.
    Der Netzwerkboot benötigt eine MVME68K BUG Version, die die NIOT und NBO Debugger kommandos unterstützt. Folge den Anweisungen in INSTALL.mvme68k wenn du mehr Details haben willst.

OpenBSD/vax:

    Boote über das Netzwerk via mopbooting, so wie es in INSTALL.vax beschrieben ist.

OpenBSD/sparc:

    Version 3.1 von OpenBSD/sparc befindet sich auf der CD3. Um von dieser CD zu booten, kannst du eines von den zwei Kommandos benutzen, die weiter unten aufgeführt sind, je nachdem welche ROM Version du hast. 

      > boot cdrom 3.1/sparc/bsd.rd
or
> b sd(0,6,0)3.1/sparc/bsd.rd

    Wenn deine sparc kein CD Laufwerk hat, kannst du alternativ dazu auch von einer Floppy booten. Dazu musst du die Datei "CD3:3.1/sparc/floppy31.fs" auf eine Floppy schreiben. Mehr Informationen gibts es in der FAQ4.1. Um von dieser Floppy zu booten, kannst du eines von den zwei Kommandos benutzen, die weiter unten aufgeführt sind, je nachdem welche ROM Version du hast. 

      > boot floppy
oder
> boot fd()

    Stelle sicher, dass du eine sauber formatierte Floppy OHNE BAD BLOCKS benutzt, ansonsten wird deine Installation mit hoher Wahrscheinlichkeit nicht klappen.

    Wenn deine sparc weder Floppy noch CD-Laufwerk hat, kannst du entweder ein bootbares Band erzeugen, oder via Netzwerk installieren, indem du den Anweisungen in der Datei INSTALL.sparc folgst.

OpenBSD/sparc64:

    Lede die CD3 in dein CDROM Laufwerk und gib boot cdrom ein.

    Wenn das nicht klappt oder du kein CDROM-Laufwerk hast, kannst du die Datei CD3:3.1/sparc64/floppy31.fs auf eine Floppy schreiben und sie mit boot floppy booten.
    Stelle sicher, dass du eine sauber formatierte Floppy OHNE BAD BLOCKS benutzt, ansonsten wird deine Installation mit hoher Wahrscheinlichkeit nicht klappen.

    You can also write CD3:3.1/sparc64/miniroot31.fs to the swap partition on the disk and boot with boot disk:b.

    Wenn nichts davon klappt, kannst du auch vom Netz booten, wie es in INSTALL.sparc64 beschrieben ist.

Hinweise zum source code:

    src.tar.gz enthält ein Source-Code-Archiv dass in /usr/src beginnt. Diese Datei enthält bis auf die Kernel-Sourcen, die sich in einem anderen Archiv befinden, alles, was du brauchst. Um es zu entpacken: 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz enthält ein Quellcode-Archiv, das in /usr/src/sys beginnt. Diese Datei enthält alle Kernel-Quellen, die man benötigt, um einen neuen Kernel zu erzeugen. Um sie zu entpacken: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Beide Archive enthalten einen regulären CVS checkout. Wenn man diese Dateibäume benutzt, ist es möglich, die anoncvs-Server wie auf http://www.OpenBSD.org/anoncvs.html zu benutzen. Wenn man diese Dateien benutzt, hat man ein viel schnelleres erstes Update, als wenn man einen frischen, eigenen OpenBSD Quellbaum (source tree) mit Hilfe von CVS holt.

Ports Tree

Mitgeliefert wird auch ein ports tree Archiv. Um es zu entpacken: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Das ports/ Unterverzeichnis ist ein checkout für den OpenBSD ports tree. Lies am besten http://www.OpenBSD.org/ports.html wenn du noch nichts über ports weisst. Dieser Text ist keine Anleitung, wie man die ports benutzt, sondern nur dazu da, um den Leuten den Einstieg in das OpenBSD ports system zu ermöglichen.

Natürlich ist das OpenBSD ports System nicht vollständig. Es ist zu bezweifeln, dass es das jemals sein wird. Trotzdem wächst es sehr schnell und wird stabiler. Fast alle ports, die ausgeliefert werden, sollten ohne Probleme auf den meisten Architekturen kompilierbar sein (auf z.B. i386 konnten über 1200 packages ohne Probleme erzeugt werden).

Das ports/ Verzeichnis stellt ein CVS (siehe in der manpage für cvs(1) nach, wenn du mit CVS nicht vertraut bist) checkout unserer ports dar. Wie unser kompletter source tree ist auch auch unser ports tree via anoncvs verfügbar. Um ihn also auf dem Laufenden zu halten, musst du den ports/ tree auf einem schreibbaren Medium haben und den tree mit diesem Kommando updaten: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_3_1

[Natürlich musst du die Namen für Server und [portsdir] mit deinen lokalen Verzeichnis, bzw. dem Namen eines anoncvs-Servers in der Nähe austauschen.]

Denk daran, dass die meisten ports via ftp als fertige packages erhältlich sind. Upgedatete packages für 3.1 werden verfügbar gemacht, sobald Probleme bekannt sind.

Falls du gerne einen weiteren port sehen willst, mithelfen willst, oder einfach mehr erfahren willst, solltest du die Mailingliste ports@openbsd.org kennenlernen.
