Dies ist die Errata- u. Patchliste für das OpenBSD-Release 2.7:

Informationen über den OpenBSD-Patchbranch gibt es hier.

Errata eines bestimmten Releases findest du hier:
2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.8, 2.9, 3.0, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 3.8, 3.9, 4.0, 4.1, 4.2, 4.2.
Du kannst auch eine tar.gz-Datei herunterladen, die alle folgenden Patches enthält. Diese Datei wird täglich auf den neuesten Stand gebracht.

Die Patches weiter unten sind mit CVS im OPENBSD_2_7-Patchbranch erhältlich.

Detailliertere Informationen darüber, wie man Patches unter OpenBSD installiert, findest du in der OpenBSD-FAQ.

Alle Architekturen

  • 040: SECURITY FIX: 18. März 2001
    Die readline-Bibliothek, die mit OpenBSD ausgeliefert wurde, erlaubt das Erzeugen von history-Dateien mit einer viel erlaubenden umask(2). Das kann zum Auslesen von sensitiven Informationen in Applikationen führen, die Passwörter und ähnliches während der Benutzerinteraktion verwenden (ein Beispiel ist mysql). Zusätzlich wird das momentane Arbeitsverzeichnis benutzt, wenn die HOME-Umgebungsvariable nicht gesetzt ist; dieser Patch schaltet die HISTORY-Datei ab, wenn HOME nicht gesetzt ist.
    Es existiert ein Quelltextpatch, der das Problem behebt.

  • 039: SECURITY FIX: 22. Februar 2001
    Es gibt eine ausnutzbare Heapcorruption in sudo.
    Es existiert ein Quelltextpatch, der das Problem behebt.

  • 037: SECURITY FIX: 4. Dezember 2000
    Der ftpd von OpenBSD 2.7 enthält einen Ein-Byte-Überlauf in der Funktion replydirname().
    Es existiert ein Quelltextpatch, der das Problem behebt.

  • 021: SECURITY FIX: 14. Juli 2000
    Unterschiedliche Probleme in den X11-Bibliotheken haben verschiedene Seiteneffekte. Wir bieten einen Jumbopatch an, um sie zu beseitigen.

    • Garstiger X-Server-DoS
      Dieser wird vom Patch für xc/programs/Xsever/os/secauth.c abgedeckt.
    • Verschiedene garstige libX11-Lücken
      Diese werden von den Patches für xc/lib/X11 abgedeckt.
    • libICE-DoS
      Dieser wird von den Patches für xc/lib/ICE abgedeckt.
    • Serverüberlauf
      Dieser wird von den Patches für xc/programs/Xserver/xkb abgedeckt.
      Die OpenBSD-Standardinstallation ist davon nicht betroffen, da der XFree86-Xwrapper bereits Tests gegen unzulässige Parameter enthält.

    Es existiert ein Quelltextpatch, der das Problem behebt.
    Hinweis 1: tcl/tk wird benötigt, um X11 aus dem Quelltext zu erzeugen.
    Hinweis 2: Beim Neuerzeugen musst du das Kommando »make DESTDIR=/ build« verwenden oder es wird einen Fehler im letzten Schritt des Builds (makewhatis) geben. Wenn du das Setzen von DESTDIR vergisst, kannst du den Buildfehler ignorieren. Die »whatis«-Datenbank wird dann beim nächsten Durchlauf von /etc/weekly neu erzeugt.

mac68k

  • 022: INSTALLATIONSFIX: 14. Juli 2000
    Der MacOS-Installer, der mit OpenBSD 2.7 ausgeliefert wird, erzeugt nicht alle notwendigen Devices, insbesondere das Device /dev/arandom nicht, das für Userland-Krypto wie z. B. ssh gebraucht wird. Das Problem tritt auf, wenn ssh-keygen bei der Erzeugung von RSA- oder DSA-Schlüsseln versagt, was zu Fehlermeldungen wie RSA-generate_keys failed oder DSA-generate_keys failed führt.
    Um das ganze zu umgehen, führe die folgenden Kommandos als root aus, sobald das System installiert wurde. 
    # cd /dev
# ./MAKEDEV arandom

    Nach diesen Kommandos (und vermutlich nach der Installation eines der ssl27-Packages), starte deine Maschine neu und sie wird ganz normal ssh-Schlüssel erzeugen.
