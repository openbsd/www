Die ist die OpenBSD 2.9 errata & Patch Liste:

OpenBSD patch branch Informationen gibt es hier.

Die Errata für eine bestimmte Version findest du hier:
2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 2.8, 3.0, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6.
Du kannst dir auch eine tar.gz Datei herunterladen, die alle folgenden Patches enthält. Diese Datei wird täglich auf den neuesten Stand gebracht.

Die Patches weiter unten sind mit CVS im OPENBSD_2_9 Patch Branch erhältlich.

Detailliertere Informationen darüber, wie man Patches installiert, findest du in der OpenBSD FAQ.

Alle Architekturen

  • 025: SECURITY FIX: 25. Juni 2002
    Ein potentieller Buffer Overflow wurde im DNS Resolver gefunden.
    Ein Source Code Patch behebt das Problem.

  • 021: SECURITY FIX: 8. Mai 2002
    Es gibt eine race condition bei der ein Angreifer file descriptor Tabelle füllen kann, die Sicherung des Kernels der fd slots 0, 1, und 2 für einen setuid oder setgid Prozess ausschalten kann.
    Ein Source Code Patch behebt das Problem.

  • 002: SECURITY FIX: 25. April 2002
    Ein Bug in sudo(8) erlaubt es einem Angreifer den Heap zu k orrumpieren, indem er einen eigenen Prompt angibt.
    Ein Source Code Patch behebt das Problem.

  • 001: SECURITY FIX: 22. April 2002
    Ein lokaler User kann super-user Privilegien wegen eines buffer overflow in sshd(8) erlangen, falls AFS auf dem System konfiguriert ist, oder auch KerberosTgtPassing oder AFSTokenPassing im sshd_config File konfiguriert ist. Ticket und token passing sind standardmäßig abgeschaltet.
    Ein Source Code Patch behebt das Problem.

  • 018: SECURITY FIX: 11. April 2002
    mail(1) erlaubt es "tilde escapes" auch im nicht-interaktiven Modus zu verarbeiten. Das kann zu einem lokalen root compromise führen.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 022: RELIABILITY FIX: 13. März 2002
    Unter bestimmten Umständen kann die zlib Kompressionsbibliothek dynamisch allozierten Speicher doppelt freigeben. Unter OpenBSD ist dies kein Sicherheitsproblem, da die BSD free(3) Funktion das entdeckt. Es gibt auch eine Kernel Zlib-Komponente, die von pppd und IPsec genutzt werden kann. Es gibt zur Zeit keine bekannte Möglichkeit den Kernel anzugreifen.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 014: SECURITY FIX: 8. März 2002
    Ein lokaler User kann Super-User Privilegien durch einen off-by-one check im channel forwarding code von OpenSSH erlangen.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 020: SECURITY FIX: 20. Februar 2002
    Eine race condition zwischen den Systemaufrufen ptrace(2) und execve(2) erlaubt einem Angreifer den Speicherinhalt von suid/sgid Prozessen zu verändern, was zu einer Kompromittierung des Super-User Accounts führen kann.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 019: SECURITY FIX: 17. Januar 2002
    Wenn der Sendmail-Ersatz Postfix auf einem System installiert ist, kann es einem Angreifer mittels sudo(8) gelingen, auf dem lokalen host root Privilegien zu erlangen. sudo ruft den Mailer als root mit einer Umgebung aufruft, die vom aufrufenden User stammt. Obwohl das ein Bug in sudo ist, ist er nach unserer Meinung nicht exploitbar, wenn Sendmail (der Mailer, der mit OpenBSD ausgeliefert wird) benutzt wird. Mit Version 1.6.5 übergibt sudo dem Mailer eine Umgebung, die nicht mehr vom aufrufenden User beeinflusst werden kann.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 017: SECURITY FIX: 28. November 2001
    Es gibt ein Sicherheitsproblem im lpd daemon, das einem Angreifer erlaubt, beliebige Dateien im root directory zu erzeugen. Nur Maschinen mit line printer access (die also in /etc/hosts.lpd oder /etc/hosts.equiv aufgeführt sind) können zu einem Angriff missbraucht werden, und der Angreifer muss auf jener Maschine auch root-Zugriff haben. OpenBSD startet den lpd in der Standard-Installation nicht.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 016: SECURITY FIX: 13. November 2001
    Es gibt ein Sicherheitsproblem im vi.recover Skript, das einem Angreifer erlauben kann, eine beliebige Datei der Länge 0 zu löschen, unabhängig davon, wem sie gehört.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 015: SECURITY FIX: 11. September 2001
    Es gibt ein Sicherheitsloch in uuxqt(8) , das einem Angreifer erlauben kann, bösartige Kommandos als User uucp auszuführen und dadurch root Zugriff zu erlangen. Der UUCP execution daemon, uuxqt(8), hat einen Bug in seinem Kommandozeilenparser , wodurch das Ausführen willkürlicher Befehle gestattet wird. Da einige UUCP Kommanados als root von cron aus ausgeführt werden (und als daemon), ist es möglich den UUCP User zu kompromittieren, um root-Rechte zu erlangen.
    Ein Source Code Patch behebt das Problem.

  • 014: SECURITY FIX: 29. August 2001
    Es gibt ein Sicherheitsloch in lpd(8) ,das einem Angreifer mit line printer access erlauben kann, root Privilegien zu erlangen. Um verwundbar zu sein, muss auf einer Maschine der lpd laufen (OpenBSD startet den lpd standardmäßig nicht). Nur Maschinen mit line printer access (ie: aufgelistet in entweder /etc/hosts.lpd oder /etc/hosts.equiv) können einen Angriff starten.
    Ein Source Code Patch behebt das Problem.

  • 013: SECURITY FIX: 21. August, 2001
    Es gibt ein Sicherheitsloch in sendmail(8) , dass einem lokalen Angreifer auf der Maschine erlaubt sich root Privilegien anzueignen, indem er debug Paramter angibt, die "out-of-bound" sind, also ausserhalb der Grenzen.
    Ein Source Code Patch behebt das Problem.

  • 012: SECURITY FIX: 30. Juli 2001
    Es gibt einen Kernel buffer overflow im NFS mount Code. Ein Angreifer könnte diesen Overflow benutzen, um bösartgien Code im Kernel-Modus auszuführen. Allerdings könnten nur User mit mount(2) Privilegien diesen Angriff starten. In der Standardinstalltion hat nur der super-user diese mount Privilegien. Die kern.usermount sysctl(3) kontrolliert, ob andere User mount Privilegien haben.
    Ein Source Code Patch behebt das Problem.

  • 011: RELIABILITY FIX: 15. Juli 2001
    Das packages(7) Subsystem akzeptiert fälschlicherweise einige package dependencies, die nicht in Ordnung sind (Details gibt's in den packages-specs(7) ).
    Ein Source Code Patch behebt das Problem, , indem er /usr/sbin/pkg dazu zwingt beim Prüfen von Versionsnummern gründlicher zu sein.

  • 008: RELIABILITY FIX: 15. Juni 2001
    Falsches Behandeln der DMA mapping Resultate durch twe(4) führt zu einem Kernel panic bei 'unaligned date transfers', wie z.B. hervorgerufen durch disklabel(8) und dump(8).
    Ein Source Code Patch behebt das Problem. Dies ist die zweite Version des Patches.

  • 007: SECURITY FIX: 15. Juni 2001
    Es gibt eine race condition in der Kernel execve(2) Implementation, die ein kleines Fenster für einen nicht-privilegierten Angreifer öffnet, der dann ptrace(2) an einen suid/sgid Prozess attachen kann.
    Ein Source Code Patch behebt das Problem.

  • 006: SECURITY FIX: 12. Juni 2001
    sshd(8) erlaubt Usern jegliche Datei namens "cookies" zu löschen, wenn X11 forwarding eingeschaltet ist. X11 forwarding ist standarmäßig ausgeschaltet.
    Es gibt einen Source Code Patch, der das Problem löst..

  • 005: RELIABILITY FIX: 7. Juni 2001
    pwd_mkdb(8) korrumpiert /etc/pwd.db wenn ein existierender User modifiziert wird.
    Es gibt einen Source Code Patch, der das Problem löst..

  • 004: RELIABILITY FIX: 5. Juni 2001
    isakmpd(8) kann kein Zertifikat mit einem Identity String benutzen, der exakt N * 8 Bytes lang ist.
    Es gibt einen Source Code Patch, der das Problem löst..

  • 003: DOCUMENTATION FIX: 1. Juni 2001
    Das 2.9 CD Cover sagt aus, dass XFree86 3.3.6-current enthalten ist. Das stimmt nur zur Hälfte. Tatsächlich ist das enhaltene XFree86 für alle die Version 4.0.2. Für den i386 ist der 3.3.6 Xserver zusätzlich enthalten, da 4.0.3 nicht alle Karten unterstützt, die unter 3.3.6 unterstützt werden.

  • 002: SECURITY FIX: 30. Mai 2001
    Programme, die die fts(3) Routinen benutzen (wie etwa rm, find, und die meisten Programme, die ein -R Flag akzeptieren) können dazu gebracht werden in das falsche Verzeichnis zu wechseln, wenn das parent dir von einem Unterverzeichnis heraus verändert wird. Das ist so ähnlich wie der alte fts Bug, passiert aber, wenn man Verzeichnisse verlässt, anstatt in sie hineinzugehen.
    Ein Source Code Patch beseitigt das Problem. Dies ist die zweite Version des Patches.

  • 001: SECURITY FIX: 29. Mai 2001
    Die signal handlers in sendmail(8) enthalten Code, der im Kontext von signal handlern unsicher ist. Das führt zu potentiell gefährlichen race conditions. Im Moment ist das nur ein theoretischer Angriff, der auch nur lokal ausgeführt werden kann (wenn überhaupt).
    Es gibt einen Source Code Patch , der das Problem dadurch beseitigt, dass er Sendmail auf Version 8.11.4 upgraded.

  • 010: RELIABILITY FIX: 9. Juli 2001
    Der nVidia Treiber für XFree86 4.0.3 setzt die Text Modus Palette falsch zurück, wenn man aus dem X Server herauskommt. Es gibt einen SourceCode Patch, der das Problem beseitigt. Damit man nicht den gesamten XFree86 Tree neu kompilieren muss, gibt es auch einen upgedateten Binärtreiber hier . Lade ihn herunter, kopiere ihn nach /usr/X11R6/lib/modules/drivers/ und starte deinen X Server neu.

  • 009: RELIABILITY FIX: 23. Juni 2001
    Das XF86Setup(1) Konfigurations-Tool für XFree86 3.3.6 produziert fehlerhafte /etc/XF86Config Dateien. Es gibt einen Source Code Patch, der das Problem löst, indem er XF86Setup gegen die XFree86 3.3.6 Version von libXxf86vm.a linkt. linkt.

  • Beim Benutzen einer PS/2 Tastatur mit einem MSI K7T Pro2A Motherboard kann es notwendig sein, die "USB Keyboard Support" und die "USB Mouse Support" Optionen im BIOS zu deaktivieren. Ansonsten wird der i8042 Keyboard Controller keine Kommandos bestätigen, was OpenBSD verwirrt.

    Die X binary sets, die mit OpenBSD 2.9 ausgeliefert wurden, sind unvollständig. Die fehlenden Dateien können per Hand nach der Installation aus den sparc tarballs hinzugefügt werden:

       # cd /; tar xzpf xbase29.tgz ./usr/X11R6/lib/X11/{rgb.txt,xdm}
   # cd /; tar xzpf xserv29.tgz ./etc/X11/xserver ./usr/X11R6/lib/X11/xserver
   # cd /usr/X11R6/bin/; ln -fs Xmac68k X
