Alle Architekturen

  • 036: SECURITY FIX: 14. November 2002
    Ein buffer overflow im named(8) kann einem Angreifer erlauben, Code mit den Privilegien des named auszuführen. Auf OpenBSD läuft der named als non-root User in einer chrooted Umgebung, was den Effekt dieses Bugs mildert.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 035: SECURITY FIX: 6. November 2002
    Ein nicht korrektes Prüfen der Argumente im getrlimit(2) Systemaufruf kann es einem Angreifer erlauben, den Kernel zum Absturz zu bringen.
    Ein Source Code Patch behebt das Problem.

  • 034: SECURITY FIX: 6. November 2002
    Ein Angreifer kann die Beschränkungen von sendmail's restricted shell smrsh(8) umgehen, und beliebige Befehle mit seinen eigenen Privilegien ausführen.
    Ein Source Code Patch behebt das Problem.

  • 033: SECURITY FIX: 21. Oktober 2002
    Ein Puffer-Überlauf kann im kadmind(8) Daemon passieren, was zu einem möglichen remote Crash oder Exploit führen kann.
    Ein Source Code Patch behebt das Problem.

  • 032: SECURITY FIX: 7. Oktober 2002
    Inkorrektes Überprüfen von Argumenten im setitimer(2) Systemaufruf kann einem Angreifer erlauben, in den Kernelspeicher zu schreiben.
    Ein Source Code Patch behebt das Problem.

  • 031: SECURITY FIX: 11. August 2002
    Ein unzureichender boundary check im select(2) Systemaufruf erlaubt einem Angreifer Kernel-Speicher zu überschreiben und beliebigen Code im Kernel-Kontext auszuführen.
    Ein Source Code Patch behebt das Problem.

  • 013: SECURITY FIX: 30. Juli 2002
    Mehrere remote buffer overflows können im SSL2 Server und SSL3 Client der ssl(8) Library auftreten, wie auch im ASN.1 Parser Code in der crypto(3) Library, alle sind potentiell remote exploitbar.
    Ein Source Code Patch behebt das Problem.

  • 012: SECURITY FIX: 29. Juli 2002
    Es gibt einen Buffer Overflow im xdr_array(3) RPC Code, was zu einem möglichen remote crash führen kann.
    Ein Source Code Patch behebt das Problem.

  • 011: SECURITY FIX: 29. Juli 2002
    Es gibt eine Race Condition im pppd(8) daemon, die ihn dazu bringen kann, eigenmächtig die Dateirechte einer anderen Datei zu verändern.
    Ein Source Code Patch behebt das Problem.

  • 010: RELIABILITY FIX: 5. Juli 2002
    Das Empfangewn von IKE payloads ausserhalb der Sequenz kann den isakmpd(8) zum Absturz bringen.
    Ein Source Code Patch behebt das Problem.
    Dies ist die zweite Version des Patches.

  • 026: SECURITY FIX: 27. Juni 2002
    Der Kernel lässt jeden User
    $OpenBSD: errata30.html,v 1.34 2004/10/29 20:14:20 jufi Exp $ Prozesse set[ug]id ausführen.
    Ein Source Code Patch behebt das Problem.

  • 025: SECURITY FIX: 25. Juni 2002
    Ein potentieller Buffer Overflow wurde im DNS Resolver gefunden.
    Ein Source Code Patch behebt das Problem.

  • 006: SECURITY FIX: 24. Juni 2002
    Alle Versionen von OpenSSH's sshd zwischen 2.3.1 und 3.3 enthalten einen Fehler bei der Eingabe-Validierung, der zu einem Integer-Overflow und "privilege escalation" führen kann. Dieses Problem ist in OpenSSH 3.4 beseitigt, und ein Patch für die angreifbaren Versionen ist als Teil des security advisory verfügbar.

  • 008: SECURITY FIX: 24. Juni 2002
    Ein buffer overflow kann während des Parsens von .htaccess im mod_ssl httpd Modul auftreten, was zu einem remote crash oder exploit führen kann.
    Ein Source Code Patch behebt das Problem.

  • 022: SECURITY FIX: 19. Juni 2002
    Ein buffer overflow kann während der Interpretation von "chunked encoding" im http daemon passieren, was zu einem möglichen remote crash oder exploit führen kann.
    Ein Source Code Patch behebt das Problem.

  • 021: SECURITY FIX: 8. Mai 2002
    Es gibt eine race condition bei der ein Angreifer file descriptor Tabelle füllen kann, die Sicherung des Kernels der fd slots 0, 1, und 2 für einen setuid oder setgid Prozess ausschalten kann.
    Ein Source Code Patch behebt das Problem.

  • 002: SECURITY FIX: 25. April 2002
    Ein Bug in sudo(8) erlaubt es einem Angreifer den Heap zu k orrumpieren, indem er einen eigenen Prompt angibt.
    Ein Source Code Patch behebt das Problem.

  • 001: SECURITY FIX: 22. April 2002
    Ein lokaler User kann super-user Privilegien wegen eines buffer overflow in sshd(8) erlangen, falls AFS auf dem System konfiguriert ist, oder auch KerberosTgtPassing oder AFSTokenPassing im sshd_config File konfiguriert ist. Ticket und token passing sind standardmäßig abgeschaltet.
    Ein Source Code Patch behebt das Problem.

  • 018: SECURITY FIX: 11. April 2002
    mail(1) erlaubt es "tilde escapes" auch im nicht-interaktiven Modus zu verarbeiten. Das kann zu einem lokalen root compromise führen.
    Es gibt einen Source-Code Patch, der das Problem behebt..

  • 017: RELIABILITY FIX: 26. März 2002
    isakmpd(8) crasht, wenn er ein IKE Paket der Länge Null empfängt, da die Überprüfung zu spät kommt.
    Es gibt einen Source-Code Patch, der das Problem behebt..

  • 016: SECURITY FIX: 19. März 2002
    Unter bestimmenten Umständen ist es auf Systemen, die YP mit netgroups in der password Database benutzen, für die rexecd(8) und rshd(8) Daemons möglich die Shell aus dem Passwort-Eintrag eines anderen Users auszuführen. Wegen eines ganz ähnlichen Problems kann atrun(8) möglicherweise in das falsche HomeVerzeichnis wechseln, wenn es at(1) jobs ausführt.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 015: RELIABILITY FIX: 13. März 2002
    Unter bestimmten Umständen kann die zlib Kompressionsbibliothek dynamisch allozierten Speicher doppelt freigeben. Unter OpenBSD ist dies kein Sicherheitsproblem, da die BSD free(3) Funktion das entdeckt. Es gibt auch eine Kernel Zlib-Komponente, die von pppd und IPsec genutzt werden kann. Es gibt zur Zeit keine bekannte Möglichkeit den Kernel anzugreifen.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 014: SECURITY FIX: 8. März 2002
    Ein lokaler User kann Super-User Privilegien durch einen off-by-one check im channel forwarding code von OpenSSH erlangen.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 013: RELIABILITY FIX: 4. Februar 2002
    Die falschen filedescriptors werden freigegeben, wenn pipe(2) versagt.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 012: SECURITY FIX: 21. Januar 2002
    Eine race condition zwischen den ptrace(2) und execve(2) Systemaufrufen erlaubt einem Angreifer den Speicherinhalt von suid/sgid Prozessen zu verändern, was zu einer Komprommitierung des super-user accounts führen kann.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 011: SECURITY FIX: 17. Januar 2002
    Wenn der Sendmail-Ersatz Postfix auf einem System installiert ist, kann es einem Angreifer gelingen auf dem lokalen host root Privilegien mittels sudo (8) zu erlangen. sudo ruft den Mailer als root mit einer Umgebung auf, die vom aufrufenden User stammt. Obwohl das ein Bug in sudo ist, ist er nach unserer Meinung nicht exploitbar, wenn Sendmail (der Mailer, der mit OpenBSD ausgeliefert wird) benutzt wird. Mit Version 1.6.5 übergibt sudo dem Mailer eine Umgebung, die nicht mehr vom aufrufenden User beeinflusst werden kann.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 010: RELIABILITY FIX: 13. Dezember 2001
    Systeme, die einen IP-in-IP Tunnel laufen haben, können mit Hilfe bösartiger Pakete zum Absturz gebracht werden.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 008: SECURITY FIX: 28. November 2001
    Es gibt ein Sicherheitsproblem im lpd daemon, das einem Angreifer erlaubt, beliebige Dateien im root directory zu erzeugen. Nur Maschinen mit line printer access (die also in /etc/hosts.lpd oder /etc/hosts.equiv aufgeführt sind) können zu einem Angriff missbraucht werden, und der Angreifer muss auf jener Maschine auch root-Zugriff haben. OpenBSD startet den lpd in der Standard-Installation nicht.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 007: SECURITY FIX: 13. November 2001
    Es gibt ein Sicherheitsproblem im vi.recover Skript, das einem Angreifer erlauben kann, eine beliebige Datei der Länge 0 zu löschen, unabhängig davon, wem sie gehört.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 006: SECURITY FIX: 13. November 2001
    pf(4) konnte nicht mit einem bestimmten Typ von IPv6 ICMP Paketen umgehen, was zu einem Crash führte.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 002: SECURITY FIX: 12. November 2001
    sshd(8) wird von OpenSSH 3.0 auf OpenSSH 3.0.2 upgegradet, um die folgenden Probleme zu beheben:

    • Ein Sicherheitsloch, das einem Angreifer erlauben könnte, sich teilweise zu authentifizieren, wenn -- und nur wenn -- der Administrator KerberosV eingeschaltet hat.
      Standardmäßig kann die OpenSSH KerberosV Unterstützung nur aktiviert werden, wenn KerberosV zuvor sauber konfiguriert wurde.

    • Ein exzessiver memory clearing Bug (von dem wir glauben, dass er nicht auszunutzen ist) existiert auch, aber da das auch zu einem Crash des Daemons führen kann, bieten wir hierfür ebenfalls einen Patch.

    • Eine Verwundbarkeit in der Übergabe der Umgebungsvariablen in der UseLogin sshd Option.

    • Verschiedene andere unkritische Fixes.

    Effektiv ein Upgrade von OpenSSH 3.0 auf OpenSSH 3.0.2, gibt es einen Sourcecode-Patch, der das Problem behebt. Das ist die zweite Version des Patches.

  • 001: INSTALL ISSUE: 12. November 2001
    Ein kleiner Bug im Installationsskript führt dazu, dass die /etc/hosts Datei falsch angelegt wird.
    Die resultierende Datei enthält eine Zeile wie die folgende:

       #.#.#.# hostname. hostname

    Die Zeile sollte eigentlich wie folgt aussehen:

       #.#.#.# hostname.domainname.com hostname

    Um das Problem zu beheben, editiere einfach die Datei und füge den Domainnamen am passenden Platz ein.
