Instalar con Otros Sistemas Operativos:

El porte de OpenBSD/i386 se puede instalar para que comparta los discos del sistema con otros sistemas operativos como MS-DOS, Windows o Linux, usando el esquema de la partición del MBR de MS-DOS y un gestor de arranque opcional. Una vez hecho esto, las operaciones normales no tienen ningún problema, pero la configuración sí que debe hacerse con cuidado ya que la instalación de Windows 95 es tristemente famosa por el poco cuidado que tiene con la información almacenada en particiones MBR ya existentes. Para más información lea las Notas de Instalación.

Medios de Instalación

Existen varias opciones para la instalación:

  • CD boot:

    El arranque desde el CD ofrece una imagen de disco de arranque de 2.88MB El Torito, que contiene casi todos los controladores para OpenBSD. Esta imagen también incluye un soporte mínimo para dispositivos USB (umass y ukbd).
    Si quiere ver una lista de los controladores más recientes disponibles en esta imagen, mire en el fichero de configuración RAMDISK_CD.

  • Floppy A:

    Esta imagen para un disquete de arranque de 1.44MB contiene los controladores más comunes. Está diseñada para cubrir las configuraciones de PC más habituales. Por lo general, los controladores que faltan son aquéllos que necesitan microcódigos grandes para funcionar, como algunas tarjetas scsi extrañas, tarjetas gigabit ethernet, o dispositivos de RAID.
    Si quiere ver una lista de los controladores más recientes disponibles en esta imagen, mire en el fichero de configuración RAMDISK.

  • Floppy B:

    Esta imagen para un disquete de arranque de 1.44MB contiene los controladores que faltan en Floppy A. Este medio está diseñado para facilitar la configuración de sistemas como servidores modernos. Contiene soporte para la mayoría de los controladores de SCSI y RAID, tarjetas gigabit ethernet, y otros. Debido a problemas de espacio, es probable que note que faltan algunos otros controladores. En particular falta el soporte para la mayoría de dispositivos PCMCIA, debido a que se trata de dispositivos muy antiguos.
    Si quiere ver una lista de los controladores más recientes disponibles en esta imagen, mire en el fichero de configuración RAMDISKB.

  • Floppy C:

    Esta imagen para disquete de 1.44MB contiene muchos controladores para dispositivos cardbus y pcmcia. Este medio está diseñado para la instalación en portátiles. En la mayoría de los casos también funcionarán con Floppy A, pero los portátiles fuera de lo corriente pueden beneficiarse más usando esta imagen.
    Si quiere ver una lista de los controladores más recientes disponibles en esta imagen, mire en el fichero de configuración RAMDISKB.

En definitiva, use el método que mejor le funcione.

Compatibilidad de Binarios con Otros Sistemas Operativos:

El porte para i386 de OpenBSD puede ejecutar binarios de Linux, FreeBSD, BSD/OS, SVR4 (incluido Solaris), y IBDS2.

Hardware con Soporte:

El porte para i386 de OpenBSD funciona en un amplio rango de PCs y clónicos de éstos, con una gran variedad de procesadores y arquitecturas con bus de E/S. Se puede instalar y ejecutar con una dificultad mínima sobre la mayoría de productos actuales. Los casos en los que se pueden encontrar problemas son, por lo general, en PC antiguos con hardware propietario, portátiles o sevidores especializados que dependan de una BIOS a medida para ocultar diferencias de implementación.

En la actualidad OpenBSD no tiene soporte para procesadores múltiples (SMP), pero funcionará usando un sólo procesador sobre una placa de un sistema de multiprocesadores.

Existe soporte para la mayoría de portátiles. Disponemos de una página en donde se detallan las características específicas de los portátiles más comunes.

La lista de hardware con soporte es de relevancia para OpenBSD-current. No se diferenciará mucho del soporte provisto en la última versión de lanzamiento (-release).

Procesadores

Todos los microprocesadores de CPU compatibles con la arquitectura Intel 80386 (i386):

  • 386/486 (SX/DX/DX2/DX4)
  • Intel Pentium/Pentium-MMX
  • Intel Pentium Pro/II/III/Celeron/Xeon
  • Intel Pentium IV
  • AMD 6x86
  • AMD K5/K6/K6-2/K6-3
  • AMD Athlon/Duron
  • Cyrix MediaGX/M1/M2
  • VIA Cyrix III
  • Rise mP6
  • IDT WinChip
  • NexGen 586
  • Transmeta TMS3200, TMS5400, TMS5600
Todo aquéllo que sea clónico del 386 ó superior debería funcionar bien. La única CPU que se sabe que funciona de forma deficiente (debido a errores de diseño de la placa base) es la Cyrix 386DLC.

Hardware Sin Soporte

El hardware que no tenga soporte puede tenerlo en un futuro; ¡su donación de tiempo, hardware o documentación puede acelerar este soporte!

  • El bus MCA "Micro Channel" que se usa en muchos modelos PS/2 de IBM.
  • Adaptadoras SCSI NCR basadas en 5380 y 53400.
  • Controladoras QIC-40 y QIC-80 (éstos son los controladores de cinta de almacenamiento que conectan al controlador de la disquetera).
  • Placas base con multiprocesadores (aunque funcionarán bien si se usa sólo un procesador). Existe un proyecto dirigido al desarrollo de este tipo de soporte.
  • Interfaces propietarias de CD-ROM Sony y Panasonic.
  • Controladoras para ZIP por puerto paralelo (las controladoras para ZIP SCSI y ATAPI funcionan bien).
  • Adaptadoras de Ethernet Intel i82556 (EtherExpress PRO/100A) e i82596 (EtherExpress PRO/10 PCI).
  • Adaptadoras de Ethernet Hewlett-Packard PC-LAN+ (HP27xxx).
  • Dispositivos de sonido SoundBlaster Live! (EMU10k1).
  • Tarjetas de sonido Aureal Vortex.
  • Winmodems.
  • Dispositivos de infrarrojos, como los que se encuentran generalmente para portátiles.
  • Firewire.
  • Controladoras de RAID Mylex e Intel.
  • PCMCIA:
    • Tarjetas de memoria.
    • La mayoría de tarjetas multifunciones.

Si está buscando probar las nuevas características de la revisión de pre-lanzamiento, puede probar una de las versiones preliminares. Para la arquitectura i386 se lanzan versiones preliminares ("snapshots") cada cierto periodo de tiempo.
