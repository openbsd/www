Introductie

Dit document is bedoeld voor mensen die -current willen volgen. Het bevat de veranderingen tussen de 3.7-uitgave en -current, en mag dus NIET worden gebruikt door iemand die 3.6 of eerder gaat upgraden, of door mensen die -stable willen volgen.

Als u wilt updaten naar de 3.7-uitgave of 3.7-stable vanaf een voorgaande versie, bekijk dan de upgrade gids.

Als u upgrade informatie wilt lezen van voorgaande OpenBSD versies, bekijk dan upgrade-old.html.

U zult ALTIJD een snapshot moeten gebruiken als beginpunt als u -current gaat gebruiken. Upgraden door uw eigen broncode te compileren wordt niet ondersteund.

2005/03/29 - Verandering van afhandeling van uitzonderingen (i386)

OpenBSD/i386 is overgestapt op DWARF voor de afhandeling van uitzonderingen. Om te upgraden, moet u eerst gcc3 bouwen en installeren en vervolgens moet u een nieuwe versie van libstdc++ bouwen en installeren: 
     # cd /usr/src/gnu/lib/libstdc++
     # make -f Makefile.bsd-wrapper cleandir
     # make -f Makefile.bsd-wrapper obj
     # make -f Makefile.bsd-wrapper
     # make -f Makefile.bsd-wrapper install

2005/03/31 - mmap(2) veranderingen(vax)

De limieten van het virtueel geheugen zijn veranderd binnen OpenBSD/vax. Vanwege de statisch-gelinkte aard van OpenBSD/vax vereist dit een speciale manier waarmee hier wordt omgegaan. De volgende stappen moeten worden genomen om een bruikbaar systeem te bouwen.
  • Update uw broncode tree
  • Bouw en installeer een nieuwe kernel, maar herstart de computer nog niet
  • Bouw en installeer een nieuwe libc: 
         # cd /usr/src/lib/libc && make depend && make && env NOMAN=1 make install
  • Bouw en installeer gcc2
  • Bouw en installeer een nieuwe make: 
         # cd /usr/src/usr.bin/make && make depend && make && make install
  • Bouw en installeer een nieuwe binutils: 
        # cd /usr/src/gnu/usr.bin/binutils
    # make -f Makefile.bsd-wrapper depend
    # make -f Makefile.bsd-wrapper
    # make -f Makefile.bsd-wrapper install
  • Herstart.
  • Bouw en installeer het systeem zoals gewoonlijk: 
        # cd /usr/src && make build

2005/04/13 - Nieuwe gebruiker en groep: _hostapd

De nieuwe, privilege gescheiden, HostAP daemon heeft een nieuwe gebruiker en groep "_hostapd" nodig. Voeg de nieuwe gebruiker en groep toe door het volgende commando als root uit te voeren: 
    useradd -u86 -g=uid -c"HostAP Daemon" -d/var/empty -s/sbin/nologin _hostapd

2005/05/27 - pf log-all wordt log (all)

Het pf.conf(5) trefwoord log-all is verwijderd. De nieuwe syntax voor deze functie is nu log (all).

2005/06/01 - mmap wordt nu meegeteld in rlimit

Anoniem geheugen geheugen dat wordt gealloceerd via mmap(2) wordt nu naar de 'datasize' bron limiet van een proces geteld. Het gevolg hiervan is, is dat het lijkt alsof processen meer geheugen in beslag nemen dan dat ze eerst deden. En oude limieten van bronnen kunnen nu te klein geworden zijn. Om dit limieten probleem op te lossen, moet u /etc/login.conf updaten: 
     # cd /usr/src/etc
     # cp login.conf /etc
log dan uit, en log weer in.

2005/06/09 - bgpd configuratie verandering

De bgpd.conf(5) trefwoorden network connected [set ...] en network static [set ...] zijn verouderd. Ze horen vervangen te worden met respectievelijk network inet connected [set ...] en network inet static [set ...].
