3.1 - Comprando um conjunto de CDs do OpenBSD

Comprar um conjunto de CDs é geralmente o melhor jeito de começar. Visite a página de pedidos para comprar sua cópia: página de pedidos do OpenBSD.

Há muitas boas razões para ter seu próprio conjunto de CDs do OpenBSD:

  • Vendas de CDs suportam o progresso do desenvolvimento do OpenBSD.
  • O desenvolvimento de um sistema operacional multiplataforma requer investimentos constantes em equipamentos.
  • Sua ajuda, na forma de uma compra do conjunto de CDs, tem uma influência real no desenvolvimento futuro.
  • Os CDs contém os binários (e fontes) para a maioria das plataformas populares suportadas.
  • Os CDs são "bootáveis" em várias plataformas, e podem ser usados para iniciar uma máquina sem um sistema operacional pré-existente instalado.
  • Os CDs são úteis para iniciar mesmo se você escolheu instalar um "snapshot".
  • Instalação pelo CD é rápida! Instalação pelo CD preserva recursos de banda da rede.
  • Os CDs do OpenBSD vêm com adesivos muito legais. Seu sistema não está completo sem eles. Você somente pode receber esses adesivos comprando um conjunto de CDs ou doando hardware.
  • O conjunto de CDs do OpenBSD vem com vários pacotes úteis e populares. O conjunto de CDs é completo o bastante para trazer um ambiente de trabalho e desenvolvimento sem qualquer conexão de rede para isso.

Se você está instalando uma versão release do OpenBSD, você deve usar um conjunto de CDs oficial.

3.3 - O OpenBSD fornece uma imagem ISO disponível para download?

A partir do OpenBSD 4.2, para determinadas plataformas, sim!

Usuários das plataformas alpha, amd64, hppa, i386, macppc e sparc64, podem agora fazer o download de uma imagem ISO, usada para criar um CD-ROM que pode inicializar e instalar todo o OpenBSD.

Lembre-se, essa ISO não é a mesma do CD oficial. Essas imagens são para plataformas únicas, e não incluem nenhum dos pacotes pré-compilados, adesivos, ou o trabalho artístico que o CD oficial inclui.

Como antes, no entanto, a instalação a partir de um arquivo ISO não é um bom método de instalação para muitas pessoas. Ainda é mais rápido e simples baixar a mídia de inicialização e então instalar apenas as partes necessárias. No entanto, para aqueles que desejam fazer um número de instalações, ou não podem colocar dez arquivos em um CD-ROM ou configurar um servidor FTP local, as ISOs estão disponíveis.

O projeto OpenBSD não disponibiliza para download as imagens ISO usadas para criar os "masters" dos CDs oficiais. A razão é simples, nós queremos que você compre o conjunto de CDs para ajudar no progresso de desenvolvimento do OpenBSD. Os direitos autorais do layout oficial do CD-ROM do OpenBSD são de Theo de Raadt. Como um incentivo para as pessoas comprar o conjunto de CDs, alguns extras são incluídos no pacote (trabalho de arte, adesivos, etc.).

Note que somente o layout do CD possui copyright, o OpenBSD em si é livre. Nada impede alguém de baixar o OpenBSD e fazer seu próprio CD.

Para aqueles que precisam de um CD bootável para seus sistemas, imagens ISO de disco de boot (chamada cd44.iso) são disponibilizadas para várias plataformas, o que permitirá que o resto do sistema seja instalado via FTP. Essas imagens ISO possuem apenas alguns poucos megabytes de tamanho, e contém apenas as ferramentas de instalação, sem os conjuntos de arquivo atuais.

3.4 - Fazendo Download via FTP, HTTP ou AFS

Existem numerosos sites de espelhos internacionais oferecendo acesso FTP e HTTP para releases e snapshots do OpenBSD. Acesso AFS também está disponível. Você deve sempre usar o site mais próximo de você. Antes de você iniciar o download de uma release ou snapshot, você pode querer usar o ping(8) e o traceroute(8) para determinar qual espelho está mais próximo de você e se ele tem um desempenho adequada. 3.5 - Selecionando o Hardware

Selecionar o hardware apropriado para rodar o seu sistema OpenBSD é importante, já que isso pode significar a diferença entre o sucesso e o fracasso de um projeto.

Se você está comprando um PC novo, seja peça por peça ou completamente pré-montado, você primeiro precisa ter certeza de que está comprando partes confiáveis. No mundo do PC, isso não é fácil. Partes ruins ou, de outra forma, não confiáveis podem fazer o OpenBSD rodar precariamente e travar frequentemente. O melhor aviso que podemos dar-lhe é ser cuidado, e comprar marcas e partes que foram revisadas por uma autoridade que você confia. Algumas vezes, uma máquina de alto custo é uma máquina de melhor qualidade. Outras vezes, esta é somente mais cara.

Existem algumas coisas que ajudarão você a obter o desempenho máxima do seu sistema:

  • Deixe a aplicação escolher o hardware: Normalmente é melhor fazer um ajuste no hardware que você estava planejando usar do que comprometer o projeto da aplicação porque você possui algo que "quer realmente usar".
  • Identifique os seus gargalos: Não gaste dinheiro com processadores poderosos se a sua aplicação é restrita à operações de I/O de disco. Não gaste dinheiro em discos rápidos se o seu sistema é limitado pela velocidade da rede. Não gaste dinheiro com mais nada se o seu gargalo é uma linha DSL de 128kbps.
  • Mantenha isso simples: Hardware simples normalmente possui problemas simples. Hardware complexo, pensado para nunca quebrar, pode render-lhe um longo tempo de conserto quando parar de funcionar.
  • Utilize hardware que você entende, ou aprenda o novo hardware antes de você implementá-lo como um sistema de produção: Independente dos méritos técnicos do hardware, usar um tipo particular de hardware antes mesmo de você se familiarizar com ele, e entender como ele funciona e como ele falha, é tolice.
  • Utilize múltiplos discos: Em vez de comprar um grande disco, compre vários discos pequenos. Embora isso custe mais, distribuir a carga em múltiplos eixos ("spindles") irá diminuir a quantidade de tempo necessário para acessar os dados nos discos. E, com mais eixos, você conseguirá um acesso mais rápido e confiável aos dados com RAID.
  • Separe os grandes blocos de armazenamento: Muitas pessoas cometem o erro de projetar um sistema que guarda grandes quantidades de dados em um Grande Bloco de Armazenamento. Normalmente esse é um projeto ruim. Você vai perceber que é melhor separar seu sistema de armazenamento em blocos gerenciáveis. Isso possui várias vantagens, duas das grandes vantagens é que você pode adicionar facilmente mais espaço de armazenamento quando você precisar (você não vai ter que estimar corretamente os requerimentos, ninguém faz isso), e que você pode comprar uma pequena quantidade de espaço para armazenamento agora, e adicionar mais quando o preço cair e a capacidade de armazenamento aumentar.
  • Evite adaptadores de rede baratos: O OpenBSD suporta vários adaptadores de rede baratos. Esses adaptadores funcionam bem em sistemas domésticos, e em ambientes de negócios ou pesquisas com transferência de dados baixa ou moderada. Mas, se você precisa de alta transferência de dados e pouco impacto no seu servidor, é melhor que você compre um adaptador de rede de qualidade. Infelizmente, alguns adaptadores de marcas renomadas não são muito melhores que os adaptadores baratos, e alguns bons adaptadores não possuem documentação completa disponível para o desenvolvimento de bons drivers. Adaptadores de Gigabit frequentemente funcionam melhor que adaptadores de 10Mbps/100Mbps, mesmo quando usados em redes de baixa velocidade, devido ao sistema de buffer superior.

3.6 - O que é apropriado como um "primeiro sistema" para aprender o OpenBSD?

Embora o OpenBSD rode em computadores pequenos, velhos e de pouca potência, diferentemente de qualquer outro SO moderno, se você está começando com o OpenBSD, usar uma máquina muito fraca pode ser frustrante. O seguinte guia é SOMENTE um guia, o OpenBSD roda muito bem em um equipamento mais modesto que o listado aqui, mas ele pode ser mais frustrante que o necessário para um usuário de primeira viagem.
  • Plataforma: Utilize uma plataforma que você está familiarizado. Quando você está aprendendo um novo sistema operacional, não é um bom momento para aprender uma nova plataforma. Assumimos que você esteja usando a plataforma i386, já que esta é, provavelmente, a que a maioria das pessoas está familiarizada.
  • Processador: Um processador Pentium 100MHz ou superior. Sim, o OpenBSD roda em um 80486 25MHz, mas você não vai querer ter todo o trabalho necessário para aprender o sistema em uma máquina lenta. A primeira irritação que você vai encontrar com um processador mais lento que esse é o tempo que ele leva para iniciar um sessão SSH na máquina. Se você deseja rodar o X, você provavelmente vai ter que usar um processador de no mínimo 200MHz. O X atualmente roda muito bem em máquinas lentas após ter sido carregado, mas ele leva um certo tempo para carregar e iniciar.
  • 32M de RAM ou mais: Se você deseja rodar o X, 128M pode ser um ponto de partida melhor.
  • Disco rígido: Um disco rígido de 1G pode ser usado para um sistema simples, como um firewall, um servidor DNS, ou algo similar. Se você deseja recompilar o sistema a partir do código, você provavelmente vai precisar de um disco de 4G, e se você deseja compilar o X também, você vai precisar de 6G ou mais. A interface IDE é recomendada para um começo. Se você tem um disco muito grande, não sinta-se obrigado a alocar todo o disco -- não há nada de errado em deixar 72G de um disco rígido de 80G desalocados se tudo que você precisa são 8G.
  • Adaptador de rede: Utilize um adaptador PCI. Mesmo que a vontade de usar um adaptador ISA antigo seja forte, resista. Você provavelmente não se lembra sobre como configurá-lo apropriadamente. Se você está planejando colocar múltiplos adaptadores de rede na máquina, escreva o endereço MAC no corpo da placa antes de colocar a tampa.
  • Boot múltiplo: Na sua primeira instalação do OpenBSD, não tente fazer boot múltiplo com outro SO. Boot múltiplo é um processo difícil de ser feito corretamente, e você deve entender bem todos os SOs envolvidos antes de tentar isso, que certamente não é o caso na sua primeira instalação. É muito possível que você apague acidentalmente todos os dados no sistema. Utilize um computador dedicado ou, no mínimo, um disco dedicado somente para isso.
  • Laptops: Enquanto muitos laptops funcionam bem com o OpenBSD, algumas vezes eles não são sistemas fáceis de fazer funcionar bem; então, um laptop pode não ser a melhor escolha para a sua primeira instalação do OpenBSD. Entretanto, uma vez que você estiver confortável com o OpenBSD, um laptop pode ser uma ferramenta muito útil.
  • Novo hardware: Hardware de marca nova, ou recém-lançado, algumas vezes não é suportado pelo OpenBSD; então, para um primeiro sistema OpenBSD, uma máquina levemente velha é recomendada.
Obviamente, mais uma coisa útil para apontar. Alguns aplicativos populares aparentemente podem usar quase todo o processador e memória que você possui no seu sistema.
