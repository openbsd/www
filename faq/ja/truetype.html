それを得るには、何をしなければいけないのでしょう ?

OpenBSD では、Xft レンダリング拡張モジュールを有効にした XFree86 バージョン 4 とともに出荷されています。 これを使えるようにするには、X サーバ (グラフィックカードドライバ) がそれをサポートして いなければなりません。いくつかのドライバは、いまだに 'fb' コードに変換されていないので、そのために、必要なレンダリング拡張モジュールを使うことができません。
次の方法でチェックしてみましょう。 
$ /usr/X11R6/bin/xdpyinfo | grep RENDER
RENDER
もし、拡張モジュールのリストの中に RENDER がなければ、誰かがそのカード用の拡張モジュールを実装するまで待つしかありません。

次の段階は、ある TrueType フォントを取得し、インストールすることです。 Xft Render 拡張モジュールの作者の Keith Packard は、TrueType フォントのパッケージを一緒に置いています。

"レッドモンドの友人のところに、(ビールの中にあるので) フリーの ナイスな TrueType フォントがあるのを見つけました。だから、それらのフォントを全部 (times、arial、comic、courier、georgia、impact、trebuchet、verdana と webdings を含む) に fonts.scale と XftConfig ファイルとを一緒にしたアーカイブを作りました。 そして、それを、次のところから取得できるようにしておきました。"

http://keithp.com/~keithp/truetype.tar.gz

あるいは、次のところから直接 (フォントごとに、ただし、fonts.dir を除く) 取得してください。
http://www.microsoft.com/typography/fontpack/default.htm

ダウンロードしたら、フォントを /usr/X11R6/lib/X11/fonts に展開しましょう。 
# cd /usr/X11R6/lib/X11/fonts
# tar xzf ~/truetype.tar.gz
もし、システム上で、これらのフォントを使いたいユーザがあなたひとりなら、 次の行を書いた $HOME/.xftconfig というファイルを作りましょう。 
	dir "/usr/X11R6/lib/X11/fonts/truetype"
システム全体でこれらのフォントを有効にしたい場合には、 /usr/X11R6/lib/X11/XftConfig を編集して、 
	dir "/usr/X11R6/lib/X11/fonts/truetype"
という行を dir "/usr/X11R6/lib/X11/fonts/Type1" の直後に追加する必要があります。

どんなアプリケーションでそれを使えるのでしょう ?

xterm、xditview、そして QT2 を使用するすべてのアプリケーション (たとえば、すべての KDE2 アプリケーション) で、Xft を利用できます。

xterm

次の設定で利用できるようになります。 
$ xterm -fa 'Andale Mono' -fs 14
option:   resource:   meaning
 -fa     *faceName    Font face name
 -fs     *faceSize    Font size
他の TrueType フォント (Arial、Georgia、Verdana、などなど) は xterm ではうまく利用できません。xterm は一文字分の範囲以上に拡張された文字を扱えるようになっていないので、文字が 1 ピクセル分ずれてしまいます。

QT2/KDE2 のアプリケーション

konqueror ブラウザのような QT2/KDE2 アプリケーションで使えるようにするには、 たったひとつ(!) の環境変数をセットするだけで良いのです。
次のように、QT_XFT を true にセットしてアプリケーションを起動するだけです。 
$ env QT_XFT=true konqueror
あるいは kdbg なら、次のようにすれば良いでしょう。 
$ env QT_XFT=true kdbg
これだけですべて OK です。
