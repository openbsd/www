3.1 - Cómo adquirir OpenBSD en CDROM

La forma más fácil para empezar a usar OpenBSD es comprando los CD. En la página de pedidos, http://www.openbsd.org/es/orders.html, se encuentra la información relacionada con la adquisición de los CD.

Existen muchas razones de peso para tener OpenBSD en CDROM:

  • La venta de los CD ayuda a la continuidad de OpenBSD.
  • El desarrollo de un sistema operativo multiplatforma requiere de una inversión constante en equipos y componentes.
  • El apoyo que supone la compra de los CD tiene un impacto real en el desarrollo de futuras versiones.
  • Los CD contienen el código binario (y el código fuente) de todas las plataformas para las que existe soporte.
  • Se puede arrancar el sistema de instalación directamente desde el CDROM en varias plataformas, y se puede usar como mecanismo de arranque de una máquina que no tenga ningún sistema operativo preinstalado.
  • Los CD son útiles como mecanismo de arranque aun cuando se haya decidido instalar una versión preliminar (snapshot).
  • ¡La instalación desde el CDROM es mucho más rápida! Instalando el sistema desde los CD se ahorran recursos de conexión a la red.
  • Los CD de OpenBSD siempre traen adhesivos curiosos. Un sistema no estará completo sin éstos. Estos adhesivos sólo se pueden conseguir comprando los CD o donando equipamiento.

Para instalar una versión final (-release) de OpenBSD se debe utilizar un CD.

3.2 - Camisetas de OpenBSD

OpenBSD produce camisetas para el disfrute de sus usuarios. Los modelos existentes se pueden ver en http://www.OpenBSD.org/es/tshirts.html. ¡A disfrutarlas! :)

3.3 - ¿Existe alguna imagen ISO compuesta por el proyecto OpenBSD?

Otros sistemas operativos de código abierto se suelen distribuir como imágenes ISO en CD-ROM. Ésta no es la forma en la que se distribuyen OpenBSD.

El proyecto OpenBSD no hace públicas las imágenes ISO que usa para producir los CD oficiales. El motivo es que nos gustaría que los usuarios adquirieran los CD, ayudando de este modo a recolectar fondos para el desarrollo de OpenBSD. Los derechos sobre la composición del CDROM oficial de OpenBSD pertenecen a Theo de Raadt. Theo no permite la redistribución de los CD oficiales de OpenBSD por parte del público. A modo de incentivo para que los usuarios compren los CD, también se incluyen algunos pequeños extras en el paquete (logotipos e imágenes, adhesivos, etc...).

Estos derechos sólo son aplicables a la composición del CD, mientras que el sistema OpenBSD en sí mismo es de libre distribución. Nada impide que alguien se baje OpenBSD y componga una imagen de CD. Si por algún motivo alguien quiere bajarse la imagen del CD, puede encontrar información a este respecto entre los archivos de las listas de correo. Cualquier imagen ISO de OpenBSD que pueda existir en Internet es una imagen no oficial, o una violación de los derechos de Theo de Raadt. La fuente de una imagen no oficial puede o no ser de fiar, y por lo tanto corresponde al usuario el determinar el grado de fiabilidad.

Aconsejamos que cualquiera que desee bajarse OpenBSD de forma gratuita, lo haga mediante la opción de instalación por FTP. Para quienes necesiten un CD de arranque para su sistema, hay disponibles imágenes ISO con disco de arranque (llamada cd33.iso) para varias plataformas que permiten que el resto del sistema se instale a través de FTP. Estas imágenes ISO sólo ocupan unos pocos megabytes, y solamente contienen las herramientas de instalación, no los archivos de instalación.

3.4 - Obtención de OpenBSD mediante FTP o AFS

Existen muchas réplicas internacionales que ofrecen acceso por FTP a las versiones de OpenBSD. También existe acceso por AFS. Es aconsejable usar siempre el servidor geográficamente más cercano. Antes de empezar a bajarse cualquier versión, es conveniente usar las utilidades ping(8) y traceroute(8) para determinar en qué servidor se encuentra la réplica más próxima, y si ésta funciona correctamente. Por supuesto que la réplica más próxima siempre será el CD de OpenBSD. A continuación puede verse la información sobre cómo acceder a OpenBSD:

http://www.openbsd.org/es/ftp.html.

3.5 - Obtención del código fuente de la versión en desarrollo

El código fuente de OpenBSD es de libre redistribución, y de disponibilidad gratuita. Por regla general, el mejor modo para empezar con un árbol de fuentes de la versión en desarrollo (-current) es instalando el código fuente desde un CD reciente, y a partir de ahí configurar AnonCVS para poder actualizar el código con regularidad. La información sobre AnonCVS, incluida la configuración, puede verse en:

http://www.openbsd.org/es/anoncvs.html.

o en la sección sobre CVS del capítulo 8 de estos documentos.

Si el ancho de banda de red del que se dispone no es suficiente para el soporte de AnonCVS, o si el acceso a Internet es por UUCP, también es posible mantener el código fuente actualizado usando CTM en lugar de AnonCVS. En tal caso es de suma importancia que se empiece desde una versión del CD. La información sobre CTM, incluida la configuración, puede verse en:

http://www.openbsd.org/es/ctm.html.

Otra alternativa es la obtención del código fuente a través de la web (http). Esto se puede hacer mediante la interfaz de cvsweb en: http://www.openbsd.org/cgi-bin/cvsweb/.
