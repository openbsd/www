OpenBSD 3.0 と OpenBSD 3.1 との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

OpenBSD 3.0 と OpenBSD 3.1 との間の変更点

  • xf86(4) の zh_CN ロケールと xim の問題を修正。
  • ELF ベースのアーキテクチャ上の Apache httpd(8) モジュールを有効化。
  • パスワードのないアカウントへの ftpd(8) ログインを不許可。
  • wdc(4) の IDE バス上のログ制御シグナルが atactl(8) から入手可能に。
  • xautolock(l) を ports(7) システム内に移動。
  • pmdb(1) の取り込み。
  • OpenBSD/macppc および powerpc の ALTIVEC サポートを改善。
  • sshd(8) で特権分離が可能に。
  • OpenBSD/i386 の CPU プローブルーチン中のオーバーフローや NULL の被参照から保護。
  • ptrace(2) の形態をカーネルの config(8) options(4) の一種に。
  • 新しい ahc(4) ドライバのいくつかの問題を修復。
  • myetheraddr() を元に戻し、hme(4) の local-mac-address プロパティ中の MAC アドレスを使用。
  • PowerPC の pmap のハンドリングを書き換え。
  • gzip(1) に与えられる拡張子が 3 文字以上である可能性、およびバッファの健全性チェックの重要性を認識。
  • identd(8) で、数字から始まるユーザ名およびグループ名を許容。
  • より多くのドライバで altq(9) をサポート。
  • zlib を、セキュリティホールが修正された 1.1.4 に更新。
  • pfctl(8) の状態表示の際の逆引きをサポート。
  • ptrace(2) でのトレース済のプロセスのメモリの読み書き用 PT_IO API を追加。
  • nc(1) が実際に持っていないポートでは listen(2) を行わない。
  • tcp(4) コードが解放済メモリ上では確実に実行されないよう変更。
  • tcp_slowtimo の代わりに timeout(9) で tcp(4) のタイマをスケジューリング。
  • uvm(9) 用に tree(3) の red-black アルゴリズムの使用を段階的に実施へ。
  • スワップを安全に使用できるようにするための raid(4) デバイス用の shutdown フックを実装。
  • crypto(4) を通じて sha1(3) および md5(3) インターフェースをエクスポート。
  • 一般的な crypto(4) および openssl(1) 関連の作業。
  • 属性の変更を無効化するための mtree(8) の保存フラグの認識。
  • sshd(8) のひとつ違い (off-by-one) エラーを修復。
  • ssh(1) のプロトコルバージョン文字列の読み出し中の不良開始メッセージからの接続クローズを区別。
  • インストーラの trap の取り扱いの多数の改良およびクリーンアップ。
  • OpenBSD/hppa のブートブロックに機種依存のコマンドを実装。
  • tcp(4) の TH_PUSH セグメントの即時 ack の挙動をトグル可能に。
  • tcp(4) サブシステム中の遅延 ack 用に、timeout(9) を tcp_fasttimo の代わりに使用。
  • pciide(4) の割り込み共有の競合状態を克服。
  • dhclient(8) の正当なリースの拒否を回避。
  • nc(1) に SOCKS4 のサポートを追加。
  • 4.4BSD の deroff(1) および spell(1) プログラムを取り込み。
  • 風変りな scsi(4) デバイスの取り扱いを拡大。
  • 意味のあるルール番号の表示による、アンロード済の pf(4) ルールのデバッグ性を改善。
  • 空の tree(3) の SPLAY_MIN および SPLAY_MAX を安全化。
  • pf(4) のオプショナルな pool(9) メモリのハードリミットをサポート。
  • 現在の pool(9) のサイズ以下への、リミットの pool_sethardlimit() による減少から保護。
  • ssh(1) のポートフォワーディングの Nagle を無効化。
  • tree(3) の splay および red-black アルゴリズムを改善。
  • ifconfig(8) が nwkey および powersave をサポート。
  • adduser(8) のグループをより徹底的に処理。
  • OpenBSD/vax 用ソースを -O2 に最適化。
  • OpenBSD/macppc 上でアパーチャドライバをサポート。
  • ユーザランドプログラムに、ハードウェア crypto(4) デバイスを利用できるようにするための kern.usercrypto を sysctl(3) インターフェースに追加。allows userland programs to utilize
  • pool(9) のオプション POOL_EXPOSE は、もはや不要なので除去。
  • extattr(9) (拡張属性) サポートの追加。
  • ktrace(2) の、その他の有用な情報の中のフォールトアドレスに加えて、siginfo_t 構造体も含むよう追加。
  • iommu ドライバのクリーンアップおよびデバッグ
  • FORK_VMNOSTACK のスタック共有コードの中のメモリレイアウトに関する欠陥のある仮定を削除。
  • pthreads(3) のシグナルを処理する際のプロセスシグナルマスクを修正。
  • netstart(8) のネットワーク構成要素の初期化順序を再編。
  • rbootd(8) の数種のシグナル競合の修正。
  • 多数のバッファのクリーンアップやアンドゥのサポートを含む、多岐にわたる mg(1) の改善。
  • pckbd(4) が有効化された場合に OpenBSD/sparc64 の beeper(4) を有効化。
  • raid(4) を少し静かに。
  • wi(4) ドライバの汎用 802.11 ioctl(2) 呼び出しのサポートを追加。
  • LP64 BE アーキテクチャの fcntl(2) F_GETOWN 問題を修正。
  • "posix" の zoneinfo に追加された、"正しい" zoneinfo timezone(3) ファイルをインストールするよう修正。
  • xebec コードの削除。
  • adduser(8) signal(3) ハンドラ中の echo(1) を有効化。
  • xargs(1) で、-x が指定されていても -n を要求しないよう修正。
  • security(8) 中で mktemp(1) を使用。
  • openssl(1) EVP API を使用するよう ssh(1) のサイファーの計算を変更。
  • デッドロックおよび MAX_KMAPENT パニックを回避するため、kmem_map-backed プールからいくつかのカーネル内メモリを割り当て。
  • mfs(8) ファイルシステム使用時の reboot(8) で起き得るパニックを回避。
  • 新しい trm(4) ドライバ。
  • pkg_install の中の、パーミッションを緩和する chmod(2) を削除。
  • ロックされたコンソールでのブロッキングを回避するため、syslog(3) 中で O_NONBLOCK を使用してコンソールを open(2)。
  • NMBCLUSTERS および BUFCACHEPERCENT、NKMEMPAGES options(4) を修正するための、config(8) がアクセス可能なフックを用意。
  • ip(4) のチェックサムを計算し、ログの完全性を保証するため、pf(4) のパケットをロギングする前に更新済ヘッダをコピーバックするよう変更。
  • kernfs(8) コードの vnode(9) のキャッシングを有効化。
  • AF_UNIX socket(2) からの/への nc(1) の接続をサポート。
  • ksh(1) での vi/emacs のファイル名補完において、ポンド記号 (#) をエスケープされた文字として解釈。
  • 新しい移植版の ahc(4) ドライバ。
  • getpw(3) および vlimit(3)、vtimes(3) 関数の削除。
  • PID 選択アルゴリズムの単純化。
  • システム時刻の変更に対して nanosleep(2) を強化。
  • pf(4) 中での時間管理に要する時間を最小化。
  • 新しい tvtohz(9) 関数。
  • pf(4) のルールのアクションならびにディレクションに skip のステップを追加、ルールセットの評価の相当の高速化を実現。
  • KAME との同期を通じた altq(9) の修正および改善。
  • sftp(1) でのファイル転送中のオーバラップした多重読み書き要求を実装。
  • apache httpd(8) 1.3.23 + mod_ssl 2.8.6 への更新。
  • ssh(1) 設定ファイルを /etc/ssh に再配置。
  • シンボリックリンクを使用した NFS の危険な過度の最適化を除去。
  • sis(4) ドライバの若干の改善。
  • heimdal-0.4e への更新。
  • FIN 再送事故の可能性があったのを修正。
  • fd(4) を共有する状況での競合を回避するための構造体ファイルへの一時参照の修正。
  • ssh(1) で -B を使用する場合に、コピーバッファ長に関する情報を含むよう変更。
  • いくつかの ssh(1) のチャネルをクリーンアップ。
  • 多相 hppa の改良の混合。
  • mpool(3) 中のバイトの並びを二重チェック。
  • 多くの新しく有益なリグレッションテストを導入。
  • sparc64 でパフォーマンスカウンタを得るためのドライバを追加。
  • ddb(4) で "boot reboot" および "boot poweroff" が可能に。
  • bha(4) ドライバを追加。
  • tcpdump(8) のヘッダのバイト交換のサポートを改善。
  • ssh(1) の "rijndael" を使用する場所で、代わりに "aes" を使用。
  • ptrace(2) に指し示すためのフラグで実行プロセスをマークするが、これはごまかしではない。
  • ubsec(4) ドライバは、DMA エラーとビッグエンディアンとをハンドリング。
  • XFree86 を 4.2.0 にアップグレード。
  • zlib、ftpd(8) および login_getcap(3) ファミリーのメモリリークを修正。
  • fdisk(8) で disklabel(8) スタイルのサイズ/オフセット値 ("300k"、"500M"、"2G" 等) をサポート。
  • いくつかの sparc64 システムに見られるシリアルコンソールドライバやキーボードを有効化。
  • pool(9) ベースの割り当て方式をシステム全体でより多く使用。
  • sendmail-8.12.2 に更新。
  • sparc64 に magma(4) および spif(4) のサポートを追加。
  • wdc(4) が、UDMA モードをサポートしていないコントローラでは、このモードを試行しないように変更。
  • pf(4) の fastroute オプションおよび route-to オプションにおける向きに注意を払う。
  • icmp(4) redirect の無視やタイムアウトを制御するための、sysctl(8) による net.inet.icmp.rediraccept および net.inet.icmp.redirtimeout を実装。
  • ssh(1) プロトコルの「未実装」メッセージのサポートと使用。
  • faithd(8)、route6d(8)、rtadvd(8) および rtsold(8) のファイルディスクリプタのクローズ失敗を防止。
  • sysctl(8) の vfs.nfs.norsvport を使用した NFS サーバの予約ポート要求を除去。
  • cs4321 ドライバの割り込みの上手な再生をハンドリング。
  • radio(4)を含む数多くのデバイスで、読み込みと書き込み動作とを正しく区別。
  • pf(4) ルール中で使用されるポート番号 0 を許容。
  • mg(1) の tty(4) のサイズ変更のサポートを改善。
  • mixerctl(1) で、本当に必要な場合には、ミキサーを RDWR でのみオープン。
  • pfctl(8) ルール解析ロジックで、正常なポートの範囲を強制。
  • POSIX に従って、0 バイト長の write(2) が実行された場合、ファイルの更新時刻を更新しない。
  • 任意のラベルによって識別される pf(4) ルールを許容。
  • hifn(4) ドライバで HiFn 7811 をサポート。
  • 他の改善項目とともに modload(8) に ELF サポートを追加。
  • rc(8) 中で savecore(8) 用のフラグをサポート。
  • nat.conf(5) の nat/rdr/binat のシンタクスの中で、変換を避けるための "no" キーワードを識別。
  • cvs(1) のコマンドラインで指定される checkout、export および update について、 タグ名による拡張を許容。
  • gif(4) を使用した ip6(4) のトンネリング上の ip(4) の振舞いを修復。
  • lkm(4) サブシステムのクリーンアップ。
  • SIGCHLD signal(3) ハンドラを無効化するため、SIG_IGN の代わりに SIG_DFL を首尾一貫して使用。
  • 認証方法がプレインテキストによるパスワードを使用しない場合でも、安全ではない tty からの root への login(1) を不許可。
  • login_chpass(8) および login_lchpass(8) を使用した、root による自身のパスワードの変更を不許可。
  • usb(3) デバイスのための XF4 による usbtablet(4) の入力サポートを追加。
  • 拒絶されたクッキーを使用した ssh(1) の X11 のチャネルがハングアップするのを回避。
  • ssh(1) プロトコル 2 の HostKey のデフォルトが /etc/ssh_host_rsa_key および /etc/ssh_host_dsa_key に。
  • sparc64 用 usb(3) デバイスを有効化。
  • 新しい m4(1) ベースの makedev(8) 生成システム。
  • fdisk(8) は MBR に書き込む際に NT のシリアル番号への参照を消去すべき。
  • FFS のファイルの穴の中間への切り詰めをハンドリング。
  • sudo(8) を 1.6.4 に更新。
  • ddb(4) にさらなるコマンドを追加。
  • sparc64 の PT_{READ,WRITE}_{I,D} を修正。
  • リグレッションテストを新しい、単一化された枠組へと移行。

  • 多くの pthreads(3) の修正: より良いタイミングでスケジューリング用のタイマ割り込みを使用する必要がある場合のみ ファイルディスクリプタをポールし、ポーリング関連のオーバーフローを回避。

  • isp(4) の大がかりなオーバーホール。

  • pf(4) の状態エントリを追加するための DIOCADDSTATE ioctl(2) を追加。
  • 他の TCP/UDP/ICMP ではないプロトコルのための、基本的なステートフルな pf(4) フィルタリングをサポート。
  • icmp6(4) MIB カウンタを修正。
  • 発生し得る競合状態を回避するための login_skey(8) のより良いシグナルの取り扱い。
  • 新たに捕捉した getpass(3) 呼び出しを補完するため、passwd(1) のシグナルハンドラを更新。
  • 2GB 長以上を生成可能とするための vnd(4) を許容。
