ユーザ

"ユーザ" という用語には、いくつかの言外の意味が含まれています。エンドユーザは、 しばしば、システムの他のユーザと会い、アイディアや問題とその解決策等を共有し、そして、 飲食を共にしながら、システムについて議論したいと考えています。これをするための最も良い 方法のひとつは、我々の世界中のユーザグループのいずれかに 参加することでしょう。

また、この用語のもうひとつの言外の意味が、"誰が何のためにシステムを使用しているのか ?" でしょう。そしてそれがこのページの残りの主題なのです。これらの会社や 組織は、OpenBSD の厳密なコード監査やセキュリティ第一の開発モデルを信頼しているのです。 彼らは、ファイアウォールや侵入検知システム、あるいは汎用のサーバを構築するために、 システムを使用しているのです。 大学の研究者や IT 部門の開発者は、しばしば、セキュリティや安定性に 関して、よく似た要求を持っていて、そのような観点から OpenBSD を選びます。また、多くの インターネットサービスプロバイダは、OpenBSD のセキュリティ機能が、 非常に強固であることを知っています。

もしあなたが、このページへの掲載をご希望される場合には、 press@openbsd.org まで、情報をお送りください。

注: セキュリティ上の理由から、会社は、彼らや彼らのクライアントたちの名前を 掲載しないようにとご依頼いただくことも可能です。その場合、彼らは "秘密の会社" として 掲載されることになるでしょう。

研究および非商用のユーザ

  • ENEP Iztacala
    ENEP Iztacala は、UNAM 附属の学校のひとつです。 UNAM はメキシコ最大の大学であり、250,000 人以上の学生が在籍し、ENEP Iztacala にも、 10,000 人を越える学生が在籍しています。ほとんどは健康関連のキャンパスであり、 コンピュータ関連のエリアは、大きなものではありません。
    私たちは、現在、OpenBSD が 2 台、Solaris が 1 台と Linux を 2 台、サーバとして運用しています。 私たちは、OpenBSD で、メインウェブサイト (7 年前の Sparcstation 5 の上で幸せに動作しています)、 そして、メールサーバ兼ファイアウォールを運用しています。
    私たちの開発エリア内には、さらに 2 台の OpenBSD コンピュータがあります。このうちの 1 台は、ネットワークモニタ (snort を使用) として稼働していて、間もなくファイアウォールの 隣に設置されるでしょう。また、もう 1 台のサーバは、OpenBSD の CVS ミラー (CVSROOT=anoncvs@anoncvs.mx.openbsd.org:/cvs) として稼働しています。
    私たちは、特にクリエイティブなものを実行しているわけではなく、OpenBSD が古い目のハードウェア でもスムーズに走り、私たちが他のオペレーティングシステムに対して抱いている、ほとんどの懸念や 疑いから私たちを解放してくれる、最高のものとして使用しているのです。
    私たちは、スペイン語の OpenBSD のメーリングリスト (openbsd@tlali.iztacala.unam.mx) も主催しています。

  • ヘルシンキ大学中央病院、フィンランド
    合計でおよそ 18,000 人ものスタッフを擁す、ヘルシンキ・ウーシマー病院地区 (HUS) は、 首都圏およびその他の構成地方自治体の居住者に対し、専門的な医療を提供しています。 OpenBSD は、学内ネットワークとインターネットの両方で、DNS、メールゲートウェイ、 VPN およびファイアウォールソリューションとして使用しています。

  • INFN イタリア核物理研究所、フィレンツェ、イタリア
    この非営利的な政府の学術研究機関は、DNS および IPF ファイアウォールパケットフィルタリング 用に OpenBSD を使用しています。

  • プラハ化学工科大学、チェコ共和国
    この大学では、WWW、教職員と学生のためのメールおよびシェルアクセスを提供するため、PC 上で OpenBSD を稼働させ、時刻サーバとセカンダリ DNS 用に SPARC IPX 上で OpenBSD を 使用しています。また、管理用ステーションとしても PC 上で OpenBSD を動かしています。

  • Sonora 州選挙管理委員会、 メキシコ
    El Consejo Estatal Electoral del Estado de Sonora usa OpenBSD para proteger sus sistemas, estas aplicaciones estan en linea a traves de Internet, dando resultados electorales al usuario, su red privada esta protegida por usuarios internos y externos.
    この政府機関では、ネットワークを保護と侵入検知を行う目的で、OpenBSD を使用しています。OpenBSD ベースの VPN は、内外のユーザに対して オンラインで選挙結果を提供しています。

  • アルバータ大学
    この大学では、プロキシサーバ、Kerberos サーバ、プリントサーバ、サービスモニタリング、 プリエンプティブなセキュリティスキャン、および問題応答のためにのため、SPARC および インテルハードウェア上で OpenBSD を使用しています。 インテルハードウェア上の OpenBSD は、大学のすべての新管理システムの向こうのセキュアな サブネットのためのファイアウォールと LAN 間の VPN に利用されています。 OpenBSD はまた、公開されているインターネットアクセスを安全に行えるようにするため、 (約 1,500 席の) キャンパスにまたがるおよそ 40 ヶ所のイーサネットジャックや、共用 研究室の前部の認証ゲートウェイ としても使用されています。 また、計算科学部では、在学生の授業のために、20 席の OpenBSD の演習室を 2 室、利用 しています。

  • ミシガン大学 情報技術統合センター (CITI)
    CITI 研究室は、多くの集中的な研究プロジェクトの ベースとして OpenBSD を使用しています。OpenBSD は、 smart card のコンテンツやプロトコル、そして、実際の個々の アプリケーションの開発や解析に使用されています。 ユーザのログインや、Kerberos のチケット獲得のための 安全なトークンを含んだカードを発行する計画が進行中です。 OpenBSD はまた、CITI における モバイルコンピューティング の実験用プラットフォームとしても利用されています。 OpenBSD は、内部的には "パケット貯蔵庫" として、 10Mbps のイーサネット LAN 上の各パケットをキャプチャーし、 CDROM に記録します。この試みは侵入検知に使用されています。 さらに、学部内の多くの人たちが、各自のメインのオペレーティング システムとして OpenBSD を使用しています。

  • ミネソタ大学
    この大学では、ネットワークモニタリングおよびキャパシティ計画を立てるために Sun Sparc ワークステーション上で OpenBSD を使用しています。それらは、(1999 年 5 月現在で) 53,000 以上の異なるインターフェースに SNMP のクエリを送り、月次処理を行うための 連結ディスクに、250MB 以上の SNMP データをロギングしています。

  • ウプサラ大学病院、感染症科
    この部署では、イントラネットサーバとして OpenBSD を使用しています。同じく、 インターネットへのファイアウォールおよびゲートウェイとしても使用しています。

  • ベルパースクール、ベルパー、ダービシャー、英国
    ベルパースクールが約 1,100 人の学生のために Samba ファイルサーバとして OpenBSD マシンを使用し、 同様に学生用ウェブホスティングおよびファイアウォール/ナットゲートウェイとしても使用しています。

  • "Forcefield" アートインスタレーション
    ニューヨークのホイットニー美術館の 2002 年の 2 年に一度の展覧会に おいて、Forcefield アートインスタレーションのための音響と照明の一部は、 OpenBSD を実行している VAXstation 3100 によって制御されています。 OpenBSD は、そのシンプルさと信頼の高さで選ばれました。

  • Warsaw 大学化学科
    この学科では OpenBSD をファイアウォール、メールサーバ、DNS サーバ、ウェブサーバ、 squid プロキシ、ファイルサーバ、あるいはその他で使用しています。そして、この学科 では、1,000 人以上の学生をサポートしています。

インターネットサービスプロバイダ

どのような ISP でも、そのひとつの目標として、顧客のサイトやアカウントを侵入の心配のない 安全な状態に保つというのがあります。OpenBSD のセキュリティの記録は、自身を雄弁に物語って いますが、このため、多くの ISP は、この理由だけでも OpenBSD を使用します。しかしながら、 他のものは、ほとんどではないにせよ、多くのサービスのために OpenBSD を使用しています。

  • Anonix
    Anonix 社は、匿名の電子メール、シェルおよびウェブホスティングサービスを提供する ISP です。これらすべてと、DNS および課金システムは OpenBSD 上で運用されています。
    「我々は、セキュリティやきれいなところ、階層化されたアプローチに、その自信を感じます。 基本インストールは巨大な不要パッケージを入れてくれたりせず、我々は、我々のシステム上の すべてがそこに属しているとの確信を持つことができる。」

  • BS Web Services
    ドイツの ISP である BS Web Services 社は、プライマリおよびセカンダリの DNS (djbdns)、 主要なウェブホスティング (Apache) と主要なメールサービス (qmail-ldap) のために OpenBSD サーバを使用しています。また、ミッションクリティカルな LDAP 認証バックエンド も MySQL データベースとともに、OpenBSD (OpenLDAP) 上で運用しています。 ホストの管理者である、Henning Brauer は次のように述べています。
    「OpenBSDは、これらのマシンではちょっとしたチューニングが必要だったが、それは特に maxprocs と maxfiles とをより大きくするというもので、これによって普通のハードウェアでも、驚くほどの負荷 状況でも処理を続けてくれるようになる。我々は、AMD Athlons (ほとんどは新しい Thunderbird) と AMD K6-III を使用している。我々は、実験用とプリントサーバ、およびあらゆる種類の他の目的のため のサーバとして、OpenBSD で運用されているいくつかの内部のマシンを持っている。我々は、特に我々の ファイアウォールのような、さらにもう少し多くのマシンを OpenBSD に移行させる計画だ。ただ、運悪く、 我々はいまでもクローズドソースのソフトウェアを使用しているのだが、これについては Linux エミュ レーションを試してみようと考えている。特に DoS アタックのような高負荷時の OpenBSD の挙動は、 我々が以前使用していた Linux とは比較にならない。だって、Linux は速度低下が著しかったのに対して、 OpenBSD では (同じハードウェアでも !) 気にせずに済んだのだから。」

    この会社は、ニューヨーク、ロサンゼルスとアムステルダムにあるデータセンタで、 WWW、FTP、電子メール、VPN トラフィックおよびネットワークモニタリングを含む 基幹業務のサービス用に OpenBSD を使用しています。 snapple.com や tanqueray.com 等のような、より大きなウェブサイトであっても、OpenBSD を変更する必要は、 特にありませんでした。
