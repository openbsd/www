OpenBSD 3.2 リリース (2002 年 11 月 1 日)

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 ここに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

他のリリースの変更点については、こちらをご覧ください。

OpenBSD 3.1 と OpenBSD 3.2 との間の変更点

  • リリースブランチを生成。
  • 3.2 のクールで新しい xdm(1) 用イメージ。
  • セキュリティのための修正: setitimer(2) システムコールの不正な境界チェックのため、攻撃者にカーネルメモリへの書き込みを許してしまうかも知れません。
    ソースコードパッチはこちら。
    [安定版への適用]
  • 古い X サーバに変更するための SIGUSR1->SIGUSR2 によるコンソール切り替えを追加実装。
  • kevent(2) のクラッシュ原因を一組修正。
  • OpenBSD 3.2-beta -> 3.2、OpenSSH -> 3.5。
  • X サーバの特権起動部分の AGP GART の初期化の試み。
  • url(4) のメモリリークを修正。
  • ライセンスの修正が完了し、login_radius が復活。
  • インストーラスクリプトの出力の修正およびさらなるクリーンアップの実施。
  • xf86cfg(1) は、サーバを '-nolisten tcp' 付きで実行。
  • xdm(1) は root として起動後、_x11 ユーザとして実行するため、特権を放棄。
  • pf(4) の双方向 NAT コードの daddr -> saddr に。やれやれ。
    [安定版への適用]
  • SSH がプローブで使用する "probe-*" に対する、ワイルドカードクライアント文字列一致を追加。
  • 厄介なライセンス問題が再発したため、login_radius 無効化。
    [安定版への適用]
  • sppp(4) および lmc(4) がより良いライセンスで復活。
  • ssh(1) は krb4 付きで、特権分離も可能に。
  • libcrypto から RC5 および MDC2 を削除。
  • setuid プログラムを含むべきではないマウントポイントに、インストーラは nosuid フラグをセットするように。
  • bridge(4) のスパニングツリープロトコルのサポート中の sizeof バグを修正。
  • Intel ギガビット Ethernet アダプタ用 gx(4) ドライバを置換するための新ドライバ em(4)。
  • talkd(8) のいくつかのメモリ割り当ておよびその他の微調整。
  • IPv6 deprecate アドレスのより良いハンドリング。
  • 巨大ペイロードオプションに先立つ IPv6 PADN オプションのパディング長を修正。
  • 32 までのどのような SSL セッション ID を許容し、従来からの非標準だった最低 16 文字制限を除去。
  • xdm(1) に /dev/X0 エントリを追加し、特権を落として立ち上がってくる xdm でのマウスの動作を許容。Upgrading Mini-faq への記載項目のひとつ。
  • netstat(1) の基底ツリーノードのダンプを正確に。
  • systrace(1) に対するテンプレートポリシーのサポート。
  • ライセンス上の問題から、sppp(4)、lmc(4) および cz(4) ドライバをリリース用カーネルから除去。
  • gcc3 に対する一連の微調整。
  • セキュリティ上の理由から、Kerberos チケットをフォワードするプログラム kf(1) および kfd(8) のビルドをしないように (3.2 のリリース後、Heimdal 0.5 がマージされれば、これは復活する予定)。
  • ベースセクションに関連してロードされる ELF セクションのサポートを追加。
  • pfctl(8) の詐称対策コマンドは、そのホストの IP アドレスのひとつをソースとしてセットされた入力パケットもブロックするように。
  • VT の切り替えコードは SIGUSR1 の代わりに SIGUSR2 を使用するように。SIGUSR1 は X サーバが xinit と同期を取るのにも使用される。
  • その場で伸長される圧縮ログファイルを、newsyslog(8) はより優雅な状況で扱うように。
  • CD に同梱のインストール作業説明書を読むのに拡大鏡を不要とするための、インストーラが生成する出力の総量の減少努力を継続中。
  • TBI (Ten-Bit Interface) モードのサポートをファイバーベースの nge(4) カードに追加し、また、他のいくつかのバグも同時に修正。
  • newsyslog(8) が実行するよう指示されたコマンドを実際に実行するように。また、ヌルコマンドの新しい解釈も追加。
  • newsyslog(8) の空行のハンドリングを修正。
  • httpd(8) から、使用されなくなった access.conf および srm.conf ファイルを削除。
  • ProxyCommand が ssh(1) の終了時に確実に終了するように (移植可能版 OpenSSH bug #223)。
  • カーネル内の初期化されていないポインタの潜在的な FREE() を修正 (sys/exec_script.c)。
  • tcpdump(8) の GRE デコーダを書き替え。
  • 実行不可能なスタックで、シグナルが飛びはねる問題を修正。
  • 冷淡で役に立たないライセンスのため、tcpdump(8) から EGP デコードのサポートを消去。
  • trsp(8) とのお別れ。
  • newsyslog(8) が特定のログファイルだけローテートすることを許容。
  • RAND_poll が /dev/arandom の代わりに、chroot された環境下で動作可能な arc4random(3) を使用するように。
  • アーカイブされたログ用に指定されたディレクトリを許容するための、新しい -a フラグを newsyslog(8) に追加。
  • kvm_open(3) の生成するファイルディスクリプタに close-on-exec フラグをセット。
  • twe(4) ドライバの DMA 関連のパニックを修正。
  • ssh(1) で、X11 のフォワーディングのリスナーソケットの生成を強固に。
  • setlocale(3) の潜在的なバッファオーバーランを修正 (NetBSD-SA2002-012)。
  • systrace(1) が deamon になる場合には、/ に chdir しない。
  • __syslog__ 文字列フォーマット属性を gcc(1) に追加。
  • systrace(1) のポリシーへの変更を定期的にセーブ。
  • newsyslog(8) へのいくつもの修正。
  • siop(4) ドライバを NetBSD と再び同期。
  • libevent のシグナルの修正。
  • Sendmail 8.12.6 をマージ。
  • atexit(3) が今受けたものと同様の mprotect() の扱いを stdio の __cleanup ハンドラに付与。
  • NAT ルールのアドレスファミリのハンドリングに対するより多くの微調整。ルールからのアドレスファミリの推測を試み、それに失敗した場合にユーザにその指定を要求。
  • cy(4) へのいくつもの修正。
  • OpenSSL-0.9.7-stable-SNAP-20020911 をマージし、libcrypto のマイナーバージョンを増加。
  • newsyslog(8) は、ログファイルのローテーションメッセージのみのログファイルのローテーションは行わないように。
  • pppd(8) のライセンス条項をほとんど最近のものに修正。
  • 存在するユーザと同様、存在しないユーザの情報を隠すための -H オプションを identd(8) に追加。NAT に便利。
  • named(8) の chroot jail から、/dev/null および /etc/localtime の必要性を除去。
  • pfctl(8) に 'antispoof' キーワードを追加。マジ。
  • pfctl(8) のネットマスクのハンドリングを改善。
  • in6_ifdetach() の失われていたポインタの初期化を追加。
  • sftp(1) クライアントの ls コマンドを便利にして、パス名展開や短かい/長いリスト表示が可能に。
  • ubsec(4) による Broadcom 582x チップの初期化の修正。
  • 多数の符合関係の修正。
  • バージョンを 3.2-beta に。
  • ssh-agent(1) は getpeereid(2) を使用してピアをチェックするように。
  • pmap_{copy,zero}_page API を変更。
  • OpenSSL 0.9.7beta3 を修正。
  • amd(8) は、127.0.0.1 および * を listen するソケットを生成。また、前者の場合、amq の要求にのみ反応。
  • pciide(4) ドライバへの Silicon Image 680 ATA133 チップのサポートを追加。
  • sshd(8) は特権分離モードで Kerberos 認証をサポート。
  • pf(4) のパーサは NAT ルールの中のアドレスファミリの解析をより注意深く行うように。
  • RFC868 モードで実行される rdate(8) への閏秒のサポートを追加 (これは、-N オプションを指定して実行される NTP モードでは既にサポート済)。
  • altq(9) の正の無限大表記を修正。
  • bootpd(8)、rtadvd(8) および rtsold(8) のシグナルハンドラを修正。
  • FD_SET のオーバーランの際、faithd(8) が落ちるように。
  • mopd(8) の一対のひとつ違い (off-by-ones) エラーを修正。
  • 長いブロックデバイスファイル名でも fsck(8) が正しく動くように (32 ではなく MAXPATHLEN 文字数分をハンドリング)。
  • デフォルトではもう、遍在する photurisd(8) 以外の何もビルドしないように。
  • 非常に多くの ANSI 化。

  • rwalld(8) は、ユーザ権限と同様、グループの権限も放棄するように。
  • safe_finger はもうインストールしないように。

  • ubsec(4) へのいくつもの互換性の修正および追加。
  • ifconfig(8) は、IPv6 を使用しないアドレスを使用するかどうかをセットできるように。
  • xdm(1) が使用するための _x11 ユーザおよびグループを追加。
  • (OpenBSD には影響を及ぼさない) 重大な Xlib のセキュリティバグ対策の XFree86 の修正を導入。
  • NAT のポートの範囲の解析を修正。
  • route-to/dup-to/fastroute が指定されたインターフェイスが実際に存在しているかどうかをチェック。これを行った場合、先に進む前にその名前を NULL ターミネイトするように。
  • ld.so(1) の初期化されていないポインタのバグを修正。
  • X サーバは、/dev/mem を試みる前に aperture ドライバのオープンを試みるように。また、i386 で早い段階で権限を落とすのを再び有効化。
  • traceroute6 と同様、DNS が相手の IP アドレスを複数返してきた場合に、traceroute(8) は警告を出力。
  • Promise Ultra133 TX2 EIDE のサポートを追加。

  • UDP の代わりに ICMP のプローブを行うよう、-I オプションを traceroute6(8) に追加。
  • setuid としてインストールされている場合に、ssh(1) が root で実行される合計時間をなお一層減少。

  • rt(4) Radiotrack ドライバを更新し、isapnp のサポートを追加。
