OpenBSD/vax

OpenBSD/vax draait op een groot aantal VAX computers die werden gemaakt door Digital (later Compaq, nu Hewlett-Packard).

Er is een mailinglijst beschikbaar voor de OpenBSD/vax port op vax@openbsd.org. Om in te schrijven op de OpenBSD/vax mailinglijst, stuur dan een bericht met als inhoud "subscribe vax" naar majordomo@openbsd.org. Lees ons mailinglijst beleid voordat je je inschrijft.

De huidige onderhouders van de port zijn Brandon Creighton en Hugh Graham. Maar anderen zijn zeker welkom om mee te helpen!

Geschiedenis:

Vanaf begin 2000 ondersteund OpenBSD ook de VAX architectuur, de code werd toen gesynchroniseerd met NetBSD's port. VAX was tweede architectruur die ooit BSD Unix heeft gedraait (in 1979!), het is wel zo netjes dat er voor deze eerbiedwaardige vaxen ook nog een modern besturingssysteem beschikbaar is.

De huidige code stamt voornamelijk af van Berkeley, met bijdragen van vrije code van Digital, Mt Xinu, NetBSD, OpenBSD en anderen.

Nadat de port werd gesynchroniseert met de code van NetBSD, werden er oplossingen en verbeteringen aangebracht, zoals ondersteuning voor 16 partities per schijf, ondersteuning voor meer VAX modellen, verbeterde emulatie van niet-geïmplementeerde instructies, enzovoorts.

Ontwikkeling zet zich nog voort en veranderingen worden zelfs vandaag de dag nog gedeelt onder de VAX hackers. Het resultaat van dit werk, is een besturingssysteem dat stabiel is en waar iedere OpenBSD gebruiker zich thuis kan voelen.

Huidige status:

Alle hardware die wordt genoemd in de ondersteunde hardware sectie kan mulit-user booten, en ondersteund genoeg genoeg ingebouwde apparatuur om over het algemeen bruikbaar te zijn. Er zijn echter enkele modellen die momenteel alleen zonder harde schijf werken.

Omdat geen enkele frame buffer van de VAXstations ondersteund wordt, zijn er op het moment geen X Window System servers beschikbaar. Echter, een complete set van X cliënten en tools zijn beschikbaar, zodat OpenBSD/vax machines dienst kunnen doen als X11 font servers of om X cliënten op afstand gebruik te laten maken van de VAX.

Echter, op het moment zitten er nog wel een aantal problemen in de OpenBSD/vax port:

  • De huidige compiler heeft soms problemen met optimaliseren (bijv.: gcc -02 genereerd onjuiste code, terwijl -01 en -00 gewoon werken). Er wordt langzaam aan gewerkt als we ze in het oog krijgen. De meeste code hoort nu gewoon te werken.
  • Het adresseren van limieten in de pmap module. Dit kan een kernel panic tot gevolg hebben. Sommige van deze situaties worden opgelost, maar andere andere kunnen misschien on-opgeloste zaken blijven.

Projecten (in willekeurige volgorde):

  • Een stuurprogramma schrijven voor VAXstation 2000's HDC9224 (floppy/MFM harde schijf controller)
  • Ondersteuning van de MSCP kan nog wat scherpe randen hebben
  • Stuurprogamma's schrijven voor de frame buffers.
  • Overstappen op ELF en gedeelde bibliotheken.
  • Ieder nieuw systeem waar we tegen aanlopen ondersteunen

Ondersteunde hardware:

Deze lijst van ondersteunde hardware is niet bedoeld om overweldigend te zijn. Vanwege een gebrek aan systemen die beschikbaar zijn voor de ontwikkelaars, zijn niet alle hardware combinaties getest.

Als gevolg hiervan, kan OpenBSD/vax misschien best wel op jouw machine werken, ook als staat het niet in deze lijst genoemt. We vertrouwen op je testen en feedback op deze lijst actueel te houden. Om ons te helpen de nauwkeurigheid van deze lijst te vergroten, stuur ALSJEBLIEFT je dmesg na de installatie naar dmesg@openbsd.org.
