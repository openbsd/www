Wat Is CVSync?

CVSync is software voor het verspreiden en updaten van source trees vanaf een master cvs(1) repository op een remote server host. De OpenBSD broncode wordt onderhouden in een CVS repository op een centrale ontwikkelings machine in Canada. Met behulp van CVSync kunnen OpenBSD ontwikkelaars gemakkelijk hun broncode actueel houden.

CVSync maakt gebruik van het zogehete "pull model" om te updaten. In het "pull model" vraagt iedere client aan de server om updates als men hier behoefte aan heeft. De server wacht op deze aangvragen van haar clients. Dus alle updates worden pas uitgevoerd als de client hier om vraagt. De server zal nooit updates sturen als de client hier niet om heeft gevraagd. De gebruiker zal óf de CVSync client handmatig moeten aanroepen om een update te krijgen, óf er moet een cron(8) job worden aangemaakt om regelmatig de updates automatisch binnen te halen.

The term CVSync verwijst naar het het gehele pakket van software. De belangrijkste onderdelen hiervan zijn de client cvsync, welke op de computer van iedere gebruiker draait, en de server cvsynd die op alle OpenBSD CVSync mirror sites draait.

CVSync is bedoeld om zowel sneller als flexibeler te zijn dan sup(1) en CVSup.

Het OpenBSD-project heeft op dit moment vijf broncode-repositories: The OpenBSD Project currently has five main source repositories:

  • src - Bevat alle broncode voor het OpenBSD Besturinssysteem.
  • ports - Bevat de OpenBSD Ports.
  • www - Bevat alle OpenBSD webpagina's (inclusief deze).
  • X11 - Bevat de OpenBSD's adaptie van het XFree86-3 software project. Deze repository heeft enkel historische waarde, de meeste gebruikers zullen geen reden hebben om hem te gebruiken, hij wordt niet langer gebruikt.
  • XF4 - Bevat OpenBSD's actieve X source tree.

Wat Is CVS?

CVS is het broncode controlesysteem dat gebruikt wordt om de OpenBSD source tree te onderhouden. Het wordt hier beter beschreven. Merk op dat CVS en CVSync totaal verschillende programma's zijn. Alhoewel CVS bedoeld is voor remote bestands toegang, is het niet ge-optimaliseerd om source trees in hun geheel over het net te verzenden. CVSync is echter wel voor bedoeld en zal dit dus ook effecienter doen.

Aan de Slag Met CVSync

Als je van plan bent om alleen maar CVSync te gaan draaien, dan wordt het aangeraden om je package van een ftp mirror te downloaden, in plaats van het zelf via de net/cvsync port te installeren.

Voor meer informatie over CVSync, zie de homepage van CVSync.

Om de OpenBSD repository middels CVSync te mirroren, kan het volgende configuratie bestand worden gebruikt: 

	config {
	    hostname anoncvs.de.openbsd.org
	    # If your network link is a T1 or faster, comment out the following line.
	    compress
	    collection {
		name openbsd release rcs
		prefix /cvs
		umask 002
	    }
	}

Dit zorgt er voor dat cvsync alle OpenBSD distributies van anoncvs.de.openbsd.org haalt met een umask(2) dat de groep schrijfrechten geeft in de lokale repository.

Er van uitgaande dat dit bestand op is opgeslagen als cvs-syncfile, dan wordt met het volgende commando cvsync aangeroepen: 

        cvsync -c cvs-syncfile

CVS Gebruiken om Je Source Tree te updaten

Het is nu makkelijk om CVSync'ed repositories op te halen. Voor de Korn/Bourne shells: 

        # cd /usr
        # CVSROOT=/cvs cvs checkout src

Voor csh en afgeleiden: 

        # cd /usr
        # setenv CVSROOT /cvs
        # cvs checkout src

Het bovenstaande zal de meest recente broncode ophalen. Velen zullen alleen de broncode willen van de uitgaves, vooral als je je systeem gaat patchen. Om deze broncode op te halen, moet je een tag meegeven met je commando. Bijvoorbeeld: 

        # cd /usr
        # cvs checkout -rOPENBSD_3_5 src

Of OPENBSD_3_4 voor 3.4, etc.

Je kan traceroute(8) gebruiken om uit te zoeken welke server het dichstbij is. Problemen met een server moeten gemeld worden aan de onderhoudeer van de server in kwestie.

Op het moment (8 december 2004), zijn dit de groottes van de CVS repositories:

openbsd-ports - 168MB
openbsd-src - 1310MB
openbsd-www - 161MB
openbsd-x11 - 200MB
openbsd-xf4 - 525MB
openbsd-cvsroot - 271MB

De gehele repository zal per jaar ongeveer 400MB in grootte toenemen.

BELANGIJKE OPMERKING: Er zijn enkele kwesties rondom cryptografische software waar iedereen bewust van moet zijn:

  • De OpenBSD broncode komt uit Canada.Zoals onderzocht door een Canadees individu en zoals beschreven in de Export Control List van Canada is het legaal om crypto-software te exporteren van Canada naar de wereld.

  • Echter wanneer je je buiten de USA of Canada bevindt moet je de cryptografische onderdelen van de OpenBSD broncode niet halen van een anoncvs-server gesitueerd in de USA. De bestanden in kwestie zijn...

    • src/kerberosIV/*
    • src/kerberosV/*
    • src/lib/libdes/*
    • src/lib/libc/crypt/crypt.c
    • src/lib/libc/crypt/morecrypt.c
    • src/sys/crypto
    • src/sys/netinet
    • src/usr.sbin/afs/src/rxkad/*
    • XF4/xc-mit/lib/Xdmcp/Wraphelp.c
    • XF4/xc-old/lib/Xdmcp/Wraphelp.c
    • XF4/xc/lib/Xdmcp/Wraphelp.c

    Vanwege de "USA ITAR munitions" lijst mag crypto-software alleen geëxporteerd worden van de USA naar Canada..
