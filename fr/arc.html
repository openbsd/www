OpenBSD/arc

OpenBSD/arc fonctionne sur les machines compatibles avec la spécification Advanced RISC Computing, connues sous le nom de machines ARC, basées sur des processeurs MIPS et initialement conçues pour Microsoft Windows NT. Ces machines incluent l'Acer PICA, "mort" depuis longtemps, ainsi que d'autres machines fabriquées par des entreprises telles que MIPS, Deskstation, NEC, et Olivetti.

La spécification ARC est éteinte, et aucune nouvelle machine à BIOS ARC pour MIPS ne sera probablement plus fabriquée. Microsoft a annoncé l'arrêt du support MIPS après NT 4.0. OpenBSD/arc fournissait une bonne alternative à NT.

Le port OpenBSD/arc a été arrêté après la sortie de la version 2.3.

A l'heure actuelle, personne ne travaille à ramener ce port à la vie. Le code source est encore disponible dans le dépôt CVS, et du code peut également être trouvé dans NetBSD. Si vous souhaitez ranimer le port arc, faites le fonctionner et contactez Per Fogelström et Warner Losh.

Historique :

Les débuts de l'histoire de ce port sont flous. Apparemment, le premier travail fut réalisé par CMU dans le cadre de leur projet Mach. Le matériel initial était des DEC Decstations à base de R2000/R3000. Ce code fut plus tard utilisé par les groupes Sprite et BSD. Le code de 4.4BSD, connu comme le port pmax, fut rendu librement disponible au courant de l'année 1993. Il fut intégré dans l'arbre de NetBSD par différentes personnes, mais sa stabilité et sa maturité ont nécessité plusieurs années de travail. Des problèmes dans les outils de compilations ont handicapé ce port, car le format d'exécutable a.out n'est pas très adapté à l'architecture MIPS.

Per Fogelström devint familier avec le code après l'avoir porté sur une carte à base de IDT R3081 faite maison. Par la suite, il ajouta le support du R4400 lorsqu'il le porta sur une carte Acer PICA avec un MIPS R4400. Willowglen Singapore fit l'acquisition d'une seconde carte PICA pour Theo de Raadt, de façon à ce qu'il puisse améliorer le port pour l'utiliser comme système de développement pour un projet interne. Depuis, Theo, Per et d'autres ont terminé ce port.

En conséquence, le code a été modifié pour le rendre plus versatile et supporter une large gamme de machines ARC. Malheureusement, la mort de cette plate-forme ainsi que le manque de disponibilité générale de ce matériel ont conduit les gens à arrêter de travailler sur ce port. Finalement, il fût décidé d'arrêter le support de ce port et de supprimer le code de l'arborescence des sources.

Matériel supporté :

Modèles supportés

  • Acer Pica (R4400PC à 150MHz , ISA, avec ethernet intégré, SCSI, video, et série)
  • Deskstation Tyne (R4600 à 133MHz , ISA et VLB)
  • Deskstation rPC44 (R4400PC à 100MHz , bus EISA)
  • Cartes d'évaluation Algorithmics R4000/R5000/R10000
  • Cartes Algorithmics P-4032 et P-5064
  • NEC RiscStation

Périphériques supportés

  • Vidéo
    • Carte graphique S3 VGA intégrée (PICA) (avec support XFree86)
    • Carte graphique standard VGA (rPC44, Tyne)
  • Clavier
    • Clavier standard compatible PC
  • Souris
    • Souris PS2 (PICA)
    • Souris série (rPC44, Tyne)
  • Ports série
    • Ports série intégrés (PICA) (ne peuvent être utilisés comme console série)
    • Contrôleurs ports série ISA
  • Ports parallèle
    • Port parallèle intégré (PICA)
    • Contrôleurs de ports parallèle ISA
  • Lecteur de disquettes
    • Contrôleur de lecteur de disquettes ISA (PICA)
  • Ethernet
    • Contrôleur ethernet SONIC intégré (PICA)
    • Cartes 3Com Etherlink
    • Cartes NE2000 compatible ISA
  • Contrôleurs SCSI
    • Contrôleur SCSI NCR53C96 intégré (PICA)
    • VLB Buslogic BT-440C/445C
    • Contrôleur SCSI ISA Buslogic BT-545 (rPC44)
  • Contrôleurs IDE
    • Contrôleurs ISA Western-Digital compatible IDE
      (uniquement bootables sur les systèmes Deskstation)
  • Périphériques divers
    • Manette sur bus ISA
    • Bus PCI sur cartes Algorithmics P-4032 et P-5064

Obtenir et installer OpenBSD/arc :

La dernière version supportée d'OpenBSD/arc et la version OpenBSD 2.3. Elle n'est plus disponible par ftp. Elle était fournie sur les CDs.
