Quoi de neuf ?

Ceci est une liste partielle des nouvelles fonctionnalités incluses dans OpenBSD 2.7. Pour une liste complètes, voir le journal des changements jusqu'à la 2.7.

  • OpenSSH supporte les protocoles SSH1 et SSH2. Le protocole SSH2 semble être lentement reconnu. Jusqu'à il y a peu de temps, SSH2 n'était disponible que dans certaines versions commerciales. Le bénéfice de SSH2 est qu'il évite d'utiliser l'algorithme de chiffrement RSA et utilise DSA à la place qui est librement réutilisable. Le problème de DSA est qu'il est lent sur les vieilles machines. Mais étant donné qu'OpenSSH supporte toujours le protocole SSH1 (si les bibliothèques RSA sont installées), nos utilisateurs peuvent utiliser le meilleur des deux mondes.

  • La manipulation astucieuse de la 2.6 est toujours là, nous permettant de distribuer le même CDROM à travers le monde contenant la crypto forte sans violer le brevet RSA aux USA. La prochaine version d'OpenBSD sera fournie avec le support RSA dans le système d'exploitation par défaut, car le brevet expire le 21 septembre 2000.

  • Beaucoup de changements à la documentation, notamment les pages du manuel et la FAQ. Les pages du manuel fournissent maintenant des exemples clairs à la place de certaines explications.

  • Une collection de "ports" plus complète et mieux testée.

  • Environ 500 packages pré-compilés et testés.

  • Le système comprend les éléments majeurs suivants venant de sources extérieures :

    • XFree86 3.3.6
    • gcc 2.95.2
    • perl 5.6.0 et quelques correctifs.
    • Apache 1.3.12 + Mod_ssl 2.6.2 + OpenSSL 0.9.5a, DSO
    • ipf 3.3.14
    • groff 1.15
    • sendmail 8.10.1
    • lynx 2.8.2 avec l'ajout du support HTTPS
    • sudo 1.6.3
    • Version récente des ncurses.
    • Le dernier KAME IPv6 et la mi-mai 2000.
    • KTH Kerberos 1.0
    • OpenSSH 2.1

  • Support périphérique grandement amélioré.

    • Support pour le matériel cryptographique pour accélérer les performances IPSEC,PowerCrypt.
    • Support pour toutes les nouveaux contrôleurs scsi Adaptec, NCR, Qlogic, et Advansys.
    • De nouveaux périphériques Ethernet sont supportés, incluant SiS900/7016, Adaptec Starfire, 21143, Sundance ST201 (DEC DFE-550TX), etc...
    • De nouveaux périphériques USB Ethernet, incluant ADMtek USB, CATC USB, et Kawasaki USB.
    • Un bon début pour le support cardbus.
    • Des améliorations significatives pour les pcmcia.
    • Support pour plus de carte réseau sans fil, comme WaveLAN, Aironet, Aviator, etc...
    • De nombreuses améliorations dans le support USB.
    • Support Hypersparc pour l'architecture sparc.
    • Support pour la plupart des cartes Ethernet gigabit sur le marché, telles que SysKonnect, Intel, et Alteon.
    • Support pour le contrôleur ICP Vortex RAID.
    • Support pour les gros PC utilisant de multiples bus PCI basés sur les chipsets Intel 451 ou RCC ServerWorks.
    • De nouveaux pilotes audio, incluant Neomagic, C-Media CMI8x38, et USB audio.
    • Améliorations dans le sous-système PCI IDE, incluant le support pour l'AMD 756.

  • Nombreuses améliorations pour la sécurité et la fiabilité (voir les lignes en rouge dans le journal des changements).

  • De nouvelles capacités, incluant :

    • Support du chiffrement de la swap.
    • Amélioration de l'émulation Linux
    • Support pour les gros systèmes de fichiers FAT32.
    • Améliorations significatives dans le code des ponts réseau (bridge).
    • Amélioration du supportpthreads.
    • Support pour les vlan.

Comment installer

Ensuite vous allez trouver les instructions qui se trouvent sur la jaquette du CDROM (http://www.OpenBSD.org/fr/orders.html). Les instructions pour effectuer une installation par FTP (ou autre) sont très semblables, les instructions pour l'installation par le CDROM sont laissées telles quelles pour vous montrer qu'il est beaucoup plus simple d'installer depuis le CDROM.

Veuillez consulter les fichiers suivants sur les CDROMs pour plus de détails quant à l'installation d'OpenBSD 2.7 sur votre machine :

  • CD1:2.7/i386/INSTALL.i386

  • CD2:2.7/sparc/INSTALL.sparc
  • CD2:2.7/mvme68k/INSTALL.mvme68k
  • CD2:2.7/amiga/INSTALL.amiga
  • CD2:2.7/mac68k/INSTALL.mac68k
  • CD2:2.7/hp300/INSTALL.hp300

  • Les versions pmax & sun3 sont disponibles sur les sites FTP mais pas sur les CD.

    • Informations pour une installation rapide, faites pour les personnes familières avec OpenBSD, et avec l'utilisation de la nouvelles commande "disklabel -E". Si vous n'êtes pas à l'aise avec l'installation d'OpenBSD, lisez les fichiers INSTALL.* correspondants cités ci-dessus.

    i386 :

    Manipuler les options du BIOS, et voyez si vous pouvez démarrez depuis le CD, en utilisant le CD1. Si ce n'est pas possible, copiez CD1:2.7/i386/floppy27.fs sur une disquette, et démarrez avec ça. Si vous désirez mélanger OpenBSD avec un autre système d'exploitation, vous aurez sans doute besoin de lire INSTALL.i386

    Pour créer une disquette depuis MS-DOS, utilisez /2.7/tools/rawrite.exe. Sous Unix, utilisez "dd if=<fichier> of=/dev/<périphérique> bs=32k" (ou le périphérique devrait être "floppy" ou "rfd0c" ou "rfd0a"). Utilisez une disquette correctement formatée SANS BLOCS DÉFECTUEUX ou vous n'y arriverez pas.

    sparc :

    Pour démarrer depuis le CD2, taper "boot cdrom 2.7/sparc/bsd.rd", ou "b sd(0,6,0)2.7/sparc/bsd.rd" suivant la version de votre ROM. Vous pouvez aussi copier CD2:2.7/sparc/floppy27.fs sur une disquette et la démarrer en utilisant "boot floppy" ou "boot fd()" suivant la version de votre ROM. Une troisième alternative est de copier CD2:2.7/sparc/kc.fs et CD2:2.7/sparc/inst.fs sur deux disquettes différentes. Insérez "kc.fs" et démarrer avec la procédure décrite ci-dessus. Une fois que la disquette s'éjecte, insérez "inst.fs". Répondez aux questions. Redémarrer depuis la disquette "kc.fs". Cette fois, quand la disquette s'éjecte, réinsérez la et répondez aux autres questions.

    amiga :

    Créer les partitions BSD suivant les instructions de la section préparation de INSTALL.amiga. Montez les CD2 sur AmigaOS en tant que périphérique CD0, ensuite exécutez la commande CLI suivante : "CD0:2.7/amiga/utils/loadbsd CD0:2.7/amiga/bsd.rd".

    hp300 :

    Vous pouvez démarrer depuis le réseau en suivant les instructions de INSTALL.hp300.

    mac68k :

    Démarrer MacOS normalement et partitioner votre disque avec les configurations A/UX appropriées. Extrayez les utilitaires Macside de CD1:2.7/mac68k/utils sur votre disque dur. Lancer Mkfs pour créer votre système de fichiers sur les partitions A/UX que vous venez de faire. Ensuite utilisez l'installeur BSD/Mac68k pour copier tous les fichiers dans CD1:2.7/mac68k/ sur vos partitions. Pour finir, vous pourrez configurer le booter BSD/Mac68k avec l'emplacement de vitre noyau et démarrer votre système.

    sun3 :

    Récupérez la version par FTP. Ensuite vous pouvez soit configurer un démarrage sans disque, soit créer une bande d'installation comme décrit dans INSTALL.sun3.

    Remarques à propos du code source :

    src.tar.gz contient une archive des sources se trouvant dans /usr/src. Ce fichier contient tout excepté les sources du noyau, qui sont dans une archive séparée. Pour l'extraire : 

    # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz contient une archive des sources se trouvant dans /usr/src/sys. Ce fichier contient toutes les sources du noyau nécessaires pour reconstruire un noyau. Pour l'extraire : 

    # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Ces deux arborescences sont régulièrement mises à jour dans le CVS. En utilisant ces arborescences, il est possible d'utiliser les serveurs anoncvs immédiatement comme décris dans http://www.OpenBSD.org/fr/anoncvs.html. Utiliser ces fichiers est beaucoup plus rapide que d'obtenir la totalité des sources depuis un serveur anoncvs.

    Arborescence des ports

    Une archive de l'arborescence des ports est aussi fournie. Pour l'extraire : 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports
# ls
...

    Le sous répertoire ports/ est une version de l'arborescence des ports OpenBSD. Veuillez lire http://www.OpenBSD.org/ports.html si vous ne savez rien à leur sujet. Ce document n'est pas un manuel sur l'utilisation des ports. C'est plus une introduction rapide sur le système des ports OpenBSD.

    Bien sur, le système des ports OpenBSD n'est pas complet. L'intégration complète des ports dans l'environnement OpenBSD est encore un projet jeune à ce jour. Nous pensons que les ports fournis sont stables mais il est plus important de réaliser que les ports vont continuer à grossir tout en apportant de nouvelles fonctionnalités dans le futur.

    Comme nous l'avons dit précédemment, les ports vont grossir dans le futur. Le répertoire ports/ représente une version CVS des ports (voir la page du manuel cvs(1) si vous n'êtes pas familier avec CVS). Comme pour l'arborescence des sources, notre arborescence des ports est disponible par anoncvs. Donc pour mettre à jour votre arborescence des ports/ depuis un media en lecture-écriture, il suffit de lancer la commande suivante : 

    # cd [répertoire des ports]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd

    [Bien sur, vous devez remplacer le répertoire local et le nom du serveur ici par l'emplacement de votre collection des port et le serveur anoncvs le plus proche.]

    Il est important de lire la page web pour les instructions spécifiques car il s'agit d'un nouveau service qui n'a pas encore été testé intensivement.

    Mais, malgré la jeunesse des ports, vous ne manquerez pas d'aide. Si vous êtes intéressé par l'ajout d'un port, voulez aider, ou voulez juste en savoir plus, la liste de discussion ports@openbsd.org est quelque chose d'utile.
