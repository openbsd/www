Historique et Etat Actuel :

Il y a un port HP PA-RISC 2.x en phase de démarrage ayant pour but de supporter les stations de travail ainsi que les serveurs 64 bit HP à base de PA8x00.

Contactez mickey si vous désirez plus d'information.

Projets (sans ordre particulier) :

  • Support hppa64 de gcc3.x (faisant maintenant partie de l'arborescence);
  • Vérifier si la version de binutils dans -current est suffisante (ce qui devrait être le cas);
  • Cloner hppa/stand pour le support 64 bit;
  • Cloner les fichiers du kernel arch/hppa pour le support 64 bit;
  • Pilotes PCI;
  • Support des systèmes multi processeurs.

Matériel supporté :

Pour l'instant il n'y a pas de matériel supporté. Voici ci-dessous, la liste des modèles pour lesquels nous prévoyons un support.

Modèles non supportés

  • A base de PA8500 : A400/rp24x0, B1000/B2000, C3000, J5000/J7000, L1000/L2000
  • A base de PA8600 : A400/rp2450, B2000/B2600, C3600, J5600, J6000, L3000/rp5400
  • A base de PA8700 : A500/rp2470, C3700, J6700/J6750, N4000/rp74x0, rp8400
  • A base de PA8800 : A500-9x, C8000, rp3410, rp3440, rp4440, rp7420, rp84x0

Cependant, les machines suivantes fonctionnent en mode 32 bit avec le port OpenBSD/hppa :

  • PA8000-based: C160, C180
  • PA8200-based: C200, C230, C240
  • PA8500-based: C360
