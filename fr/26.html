Pour obtenir les fichiers de cette version :
  • Commander un CDROM depuis notre système de commandes. Si vous voulez qu'un libraire près de chez vous vous l'obtienne, demandez lui de commander
    ISBN 0-9683637-4-1.
  • Voir les informations sur la page FTP pour une liste des machines miroirs
  • Aller dans le répertoire pub/OpenBSD/2.6/ sur l'un des sites miroirs
  • Lisez brièvement le reste de ce document.
  • Jetez un oeil sur la page des erratas 2.6 pour une liste des bugs et des solutions.



Tous les copyrights applicables et les différents crédits se trouvent dans les fichiers sources se trouvant dans src.tar.gz et srcsys.tar.gz.

Ensuite vous allez trouver les instructions qui se trouvent sur la jaquette du CDROM (http://www.OpenBSD.org/orders.html). Les instructions pour effectuer une installation par FTP (ou autres) sont très semblables, les instructions pour l'installation par le CDROM sont laissées telles quelles pour vous montrer qu'il est beaucoup plus simple d'installer depuis le CDROM.

Veuillez consulter les fichiers suivants sur les CDROMs pour plus de détails quant à l'installation d'OpenBSD 2.6 sur votre machine :

  • CD1:2.6/i386/INSTALL.i386
  • CD1:2.6/sparc/INSTALL.sparc

  • CD2:2.6/alpha/INSTALL.alpha
  • CD2:2.6/amiga/INSTALL.amiga
  • CD2:2.6/mac68k/INSTALL.mac68k
  • CD2:2.6/hp300/INSTALL.hp300

  • Une version pmax est disponible sur les sites FTP mais pas sur les CD.

    • Informations pour une installation rapide, faites pour les personnes familières avec OpenBSD, et avec l'utilisation de la nouvelle commande "disklabel -E". Si vous n'êtes pas à l'aise avec l'installation d'OpenBSD, lisez les fichiers INSTALL.* correspondants cités ci-dessus.

    i386 :

    Manipuler les options du BIOS, et voyez si vous pouvez démarrer depuis le CD, en utilisant le CD1. Si ce n'est pas possible, copiez CD1:2.6/i386/floppy26.fs sur une disquette, et démarrez avec ça. Si vous désirez mélanger OpenBSD avec un autre système d'exploitation, vous aurez sans doute besoin de lire INSTALL.i386.

    Pour créer une disquette depuis MS-DOS, utilisez /2.6/tools/rawrite.exe. Sous Unix, utilisez "dd if=<fichier> of=/dev/<périphérique> bs=32k" (ou le périphérique devrait être "floppy", "rfd0c" ou "rfd0a"). Utilisez une disquette correctement formatée SANS BLOCS DÉFECTUEUX ou vous n'y arriverez pas.

    sparc :

    Pour démarrer depuis le CD1, taper "boot cdrom 2.6/sparc/bsd.rd", ou "b sd(0,6,0)2.6/sparc/bsd.rd" suivant la version de votre ROM. Vous pouvez aussi copier CD1:2.6/sparc/floppy26.fs sur une disquette et la démarrer en utilisant "boot fd()" ou "boot floppy" suivant la version de votre ROM. Une troisième alternative est de copier CD1:2.6/sparc/kc.fs et CD1:2.6/sparc/inst.fs sur deux disquettes différentes. Insérez "kc.fs" et démarrez avec la procédure décrite ci-dessus. Une fois que la disquette s'éjecte, insérez "inst.fs". Répondez aux questions. Redémarrer depuis la disquette "kc.fs". Cette fois, quand la disquette s'éjecte, réinsérez la et répondez aux autres questions.

    amiga :

    Créer les partitions BSD suivant les instructions de la section préparation de INSTALL.amiga. Montez le CD2 sur AmigaOS en tant que périphérique CD0, ensuite exécutez la commande CLI suivante : "CD0:2.6/amiga/utils/loadbsd CD0:2.6/amiga/bsd.rd".

    hp300 :

    Vous pouvez démarrer depuis le réseau en suivant les instructions de INSTALL.hp300.

    alpha :

    Votre alpha doit utiliser un firmware SRM (pas ARC). Si vous avez un CDROM, vous pouvez essayer "boot -fi 2.6/alpha/bsd.rd dkaX" (utilisez "show device" pour trouver l'identifiant de votre lecteur de CDROM). Sinon, copier CD2:2.6/alpha/floppy.fs sur une disquette et démarrer la en tapant "boot dva0". Si cela ne fonctionne pas, vous pouvez placer bsd.rd sur un autre périphérique et le démarrer, ou utiliser le simpleroot fourni.

    mac68k :

    Démarrez MacOS normalement et partitionez votre disque avec les configurations A/UX appropriées. Extrayez les utilitaires Macside de CD1:2.6/mac68k/utils sur votre disque dur. Lancez Mkfs pour créer votre système de fichiers sur les partitions A/UX que vous venez de faire. Ensuite utilisez l'installeur BSD/Mac68k pour copier tous les fichiers dans CD1:2.6/mac68k/ sur vos partitions. Pour finir, vous pourrez configurer le booter BSD/Mac68k avec l'emplacement de votre noyau et démarrer votre système.

    Remarques à propos du code source :

    src.tar.gz contient une archive des sources se trouvant dans /usr/src. Ce fichier contient tout excepté les sources du noyau, qui sont dans une archive séparée. Pour l'extraire : 

    # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz contient une archive des sources se trouvant dans /usr/src/sys. Ce fichier contient toutes les sources du noyau nécessaires pour reconstruire un noyau. Pour l'extraire : 

    # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Ces deux arborescences sont régulièrement mises à jour dans le CVS. En utilisant ces arborescences, il est possible d'utiliser les serveurs anoncvs immédiatement comme décrit dans http://www.OpenBSD.org/fr/anoncvs.html. Utiliser ces fichiers est beaucoup plus rapide que d'obtenir la totalité des sources depuis un serveur anoncvs.

    Arborescence des ports

    Une archive de l'arborescence des ports est également fournie. Pour l'extraire : 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports
# ls
...

    Le sous répertoire ports/ est une version de l'arborescence des ports OpenBSD. Veuillez lire http://www.OpenBSD.org/ports.html si vous ne savez rien à leur sujet. Ce document n'est pas un manuel sur l'utilisation des ports. C'est plus une introduction rapide sur le système des ports OpenBSD.

    Bien sur, le système des ports OpenBSD n'est pas complet. L'intégration complète des ports dans l'environnement OpenBSD est encore un projet jeune à ce jour. Nous pensons que les ports fournis sont stables mais il est plus important de réaliser que les ports vont continuer à grossir tout en apportant de nouvelles fonctionnalités dans le futur.

    Comme nous l'avons dit précédemment, les ports vont grossir dans le futur. Le répertoire ports/ représente une version CVS des ports (voir la page du manuel cvs(1) si vous n'êtes pas familier avec CVS). Comme pour l'arborescence des sources, notre arborescence des ports est disponible par anoncvs. Donc pour mettre à jour votre arborescence des ports depuis un media en lecture-écriture, il suffit de lancer la commande suivante : 

    # cd [répertoire des ports]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd

    [Bien sur, vous devez remplacer le répertoire local et le nom du serveur ici par l'emplacement de votre collection des ports et le serveur anoncvs le plus proche.]

    Il est important de lire la page web pour les instructions spécifiques car il s'agit d'un nouveau service qui n'a pas encore été testé intensivement.

    Mais, malgré la jeunesse des ports, vous ne manquerez pas d'aide. Si vous êtes intéressé par l'ajout d'un port, voulez aider, ou voulez juste en savoir plus, la liste de discussion ports@openbsd.org vous sera utile.

    QUOI DE NEUF

    Ci-dessous se trouve une liste partielle des nouvelles fonctionnalités incluses dans OpenBSD 2.6. Pour une liste complète, voir le journal des changements jusqu'à la 2.6.

    • L'ajout le plus intéressant est OpenSSH (http://www.OpenSSH.com/). Il s'agit d'une version libre et réutilisable de SSH basée sur une version ancienne écrite par Tatu Ylonen (1.2.12). Cette version fut la dernière avec une licence libre. OpenSSH a été mis à jour jusqu'aux standards actuels et utilise la bibliothèque OpenSSL. Il est libre sauf pour les utilisateurs commerciaux aux USA (à cause du brevet RSA aux USA). OpenSSH a été développé par OpenBSD et a été porté sur FreeBSD et Linux.
    • Une manipulation astucieuse nous permet de distribuer le même CDROM (USA et reste du monde) et de maintenir toute la force de la cryptographie sans violer le brevet RSA aux USA.
    • Beaucoup de changements à la documentation, notamment les pages du manuel et la FAQ. Les pages du manuel fournissent maintenant des exemples clairs à la place de certaines explications. En gardant la documentation concise et en deux endroits différents, nous espérons éviter l'explosion des documents de type "How-To" qui ne permettent pas de centraliser l'information.
    • Une collection des "ports" plus complète. Les ports sont la méthode à utiliser pour télécharger et compiler des applications libres depuis le réseau (applications, lecteurs de mail, navigateurs, etc...). Les utilisateurs ont juste besoin d'aller dans le répertoire correspondant et de taper "make install" pour démarrer un processus qui va récupérer les sources, les corriger pour OpenBSD, compiler et installer le package. La plupart des ports sont aussi disponibles en packages pré-compilés.
    • Inclus :
      • XFree86 3.3.5
      • gcc 2.95
      • Apache 1.3.9 + Mod_ssl 2.4.5 + OpenSSL 0.9.4, et support DSO
    • De nombreuses améliorations pour la sécurité et la fiabilité (voir les lignes en rouge dans la liste complète)
    • De nouveaux périphériques matériels sont supportés, notamment sur l'architecture PC (i386).
      • support PCI IDE et DMA
      • USB (universal serial bus)
      • Périphériques ATAPISCSI, incluant CD-R et CD-RWs
