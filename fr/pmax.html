OpenBSD/pmax

État et histoire :

Des machines pmax sont disponibles pour les développeurs, demandez à Dieter Mayer .

Le port pmax d'OpenBSD a été démarré par Thomas Graichen (graichen@OpenBSD.org) juste avant la version 2.0 d'OpenBSD. Pour l'instant il n'y a pas de mainteneur. Si vous voulez maintenir ce port, il y a du matériel disponible, donc envoyez un message. Le code spécifique pmax est fortement basé sur NetBSD/pmax. Je voudrai pouvoir remercier ici Dieter Mayer et Rob Urban de m'avoir (Thomas) fournit le matériel pour faire le développement. De plus, je voudrai remercier les personnes de l'équipe NetBSD/pmax et des listes de discussion, spécialement Jonathan Stone et Michael Hitch pour leur aide dans de nombreuses situations. Si vous êtes intéressé par l'obtention d'OpenBSD/pmax, vous pouvez utiliser la dernière version d'OpenBSD ou l'un des snapshots OpenBSD/pmax si vous êtes intéressé pour suivre -current. Vous pouvez obtenir cela depuis nos serveurs FTP OpenBSD habituels.

En dépit du fait que le code spécifique au matériel dans OpenBSD/pmax n'est pas très différents de NetBSD/pmax ce n'est pas le cas pour le système dans son ensemble...

Différences avec NetBSD/pmax :

  • le port est son propre hôte (... cela signifie qu''il utilise les outils fournis, il n'y a pas besoin d'avoir un binutils externe).
  • vous pouvez obtenir toutes les améliorations d'OpenBSD, ajouts et correctifs - le plus important étant les correctifs et les ajouts liés à la sécurité

Machines supportées :

OpenBSD/pmax supporte actuellement les machines suivantes :

  • decstation 2100 and 3100 (pmin et pmax)
  • decstation 5000/120, /125 et /133 (3min)
  • decstation 5000/200 (3max)
  • decstation 5000/240 (3maxplus)
  • decstation 5000/20 and /25 (personal decstation)

OpenBSD/pmax ne fonctionne pas sur :

  • decstation 5100 (cousin de la decstation 3100 basé sur le r2000)
  • decsystem 5400 et 5500 (proche des vax mayfair et mayfair II avec des cpu mips)
  • decsystem 5800 (proche des vax 6300/6300 avec des cpu mips)

Per Fogelstrom (pefo@OpenBSD.org) a maintenant une decstation upgradée avec un cpu r4000 ... donc je m'attends à un support pour celles-ci dans le futur.

Périphériques supportés :

Si quelque chose est faux ici - veuillez me le signaler !

Framebuffer :

  • pm mono (1024x864x1) ou couleur (1024x864x8) onboard framebuffer (pmin et pmax)
  • PMAG-DV (1024x864x8) onboard framebuffer (personal decstation)
  • Carte PMAG-AA (1280x1024x1) turbochannel framebuffer - mfb
  • Carte PMAG-BA (1024x864x8) turbochannel framebuffer - cfb
  • Carte PMAGB-BA (1280x1024x8) turbochannel framebuffer - sfb

    • Souvenez vous que vous aurez besoin d'un moniteur qui peut se synchroniser sur le vert parce que les framebuffers envoient la synchronisation sur la sortie verte. Le moniteur devrait aussi être multisync ou alors avoir un fréquence fixe compatible dec. Tous les framebuffers ci-dessus devraient être supportés par le serveur x.

Ethernet :

  • Interface amd lance ethernet sur carte
  • Carte d'interface turbochannel amd lance lance

SCSI :

  • Contrôleur sur carte dec SCSI - sii (pmin et pmax)
  • Contrôleur sur carte dec SCSI - asc
  • Carte contrôleur turbochannel dec SCSI - asc

Divers :

  • Clavier dec lk201 ou compatible
  • souris dec ou compatible
  • ports séries sur carte sur la plupart des machines
