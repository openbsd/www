Changes made between OpenBSD 7.9 and -current

  • Mitigate floating point state leakage observed on AMD Zen/Zen+ (Zen 1).
  • Attach puc(4) in RAMDISK_CD.
  • Fix parking mutex on sun4u sparc64 cpus.
  • Copy SpacemiT K1 device trees onto the riscv64 miniroot.
  • Keep malicious or confused scsi tape (st(4)) devices from causing reading or writing outside a mode sense/select buffer.
  • Fix vmd(8) vionet reset race leading to broken networking.
  • Floppy install users on i386/amd64 may find fw_update(8) for some drivers will not work because pci strings in the kernel have become too large.
  • Prepare for refining unveil(2) usage by providing a function manpath_unveil() that makes the manpath directories accessible.
  • Disallow loading if mapchar emulops require a question mark character that is missing.
  • In ipcs(1) use shm_cpid to determine whether a shared memory segment is in use.
  • Make vio.4 recover from missed RX interrupts.
  • Bump smtpd to version 7.9.
  • Prevent potential slaacd crash if an attacker on the same layer 2 network sends rogue router advertisements.
  • For sysctl({CTL_KERN, KERN_TTY, KERN_TTY_INFO), only export the t_session kernel address pointer if the caller is root.
  • Do not expose p_addr kernel address unless root.
  • Fix mapping of kernel l1 page table entries on riscv64.
  • Start forked children without a pgrp (i.e. NULL) and update the pgrp pointer late to fix a potential race.
  • Initialize GuC TLB invalidation xarray with XA_FLAGS_LOCK_IRQ to avoid a "locking against myself" panic seen on a non-mp kernel using i915 drm(4).
  • Add smte(4), a driver for the ethernet interfaces of the SpacemiT K1 SoC.
  • Add -C flag to tmux(1) command-prompt to match display-message -C.
  • Add the duration of the current ssh(1) session to the information displayed with the ~I escape.
  • Fix a hang in rad(8) and slaacd(8) when they receive an RA from the local network with an ND option of length zero.
  • Fix a memory leak on sensorsd(8) configuration reload.
  • Fix a possible race in sparc64 ipi_drop_fpstate().
  • Initialize all FPU registers on sparc64 to all 1 (or -NaN) not only the lower 32.
